    # 总结

    # 认证

    目标：得到认证对象，存储到Security的上下文对象中

    SecurityContextHolder：上下文对象工具类，采用ThreadLocal 实现，保证获取当前用户的上下文对象。

    SecurityContext：上下文对象，核心对象，封装很多当前用户信息对象

    # 认证过滤器

    框架提供了以下几种：

    • 用户密码认证：AbstractAuthenticationProcessingFilter抽象类和UsernamePasswordAuthenticationFilter实现类
    • 匿名认证：AnonymousAuthenticationFilter
    • Basic认证：BasicAuthenticationFilter
    • Digest认证：DigestAuthenticationFilter
    • RememberMe认证：RememberMeAuthenticationFilter
    • Session认证：SessionManagementFilter

    参考 GenericFilterBean类的子类XXAuthenticationFilter，自定义扩展认证过滤器，可以参考命名

    # 用户密码认证

    • 认证对象：即Authentication接口，最终的封装对象，isAuthenticated()方法判断是否认证成功
    • 登录对象：即UserDetails接口，包含登录信息，用于登录验证
    • 认证管理器：即AuthenticationManager接口，默认实现类ProviderManager
    • 认证程序：即AuthenticationProvider接口，常用实现类DaoAuthenticationProvider，其抽象父类AbstractUserDetailsAuthenticationProvider

    大致步骤如下：

    1. 过滤器拦截（doFilter），先判断是否是需要认证的请求，再获取认证对象
    2. 实现类则创建认证对象，用认证管理器认证
    3. 默认实现类，遍历内部属性认证程序集合认证
    4. 遍历认证，认证程序是否支持认证？认证？有一种能返回认证对象即可
    5. 比如用户密码的认证程序？
      1. 从缓存获取登录对象
      2. 根据UserDetailsService接口的loadUserByUsername()获取登录对象
      3. 验证登录对象，根据接口的4个方法验证是否有效
      4. 创建认证对象，返回
    6. 存储在SecurityContext上下文对象中

    # 开发参考

    通过以上步骤了解到，实际开发常见有以下几种方式：

    • 自定义过滤器，继承OncePerRequestFilter抽象类，结果Authentication直接存储SecurityContext上下文对象中。适合扩展如JWTAuthenticationFilter
    • 自定义认证程序，实现AuthenticationProvider接口，重写authenticate()，返回Authentication。适合扩展如短信、验证码方式
    • 自定义UserDetailsService实现类，重写loadUserByUsername()，可扩展UserDetails实体类，返回UserDetails对象。常规扩展，重写登录对象的获取方法

    # 鉴权

    目标：请求URL通过鉴权可以访问

    # 鉴权过滤器

    • FilterSecurityInterceptor：默认顺序是所有安全过滤器中的最后一个
    • MethodSecurityInterceptor：适用于方法鉴权注解拦截

    # 默认过滤器

    FilterSecurityInterceptor过滤器大致步骤如下：

    • 过滤器链对象：即FilterInvocation类，spring的所有过滤器也是基于一个过滤器链去逐个doFilter
    • 访问决策管理器：即AccessDecisionManager接口，默认实现类AffirmativeBased
    • 安全元数据对象：即SecurityMetadataSource接口，默认子接口FilterInvocationSecurityMetadataSource，默认实现类DefaultFilterInvocationSecurityMetadataSource
    • 权限对象集合：即Collection<ConfigAttribute> configAttributes
    • 投票决策：即AccessDecisionVoter接口，默认实现类WebExpressionVoter

    1. 过滤器拦截（doFilter），请求封装成过滤器链对象

    2. 判断是否要触发前置后置验证，或直接验证过滤器链

    3. 需要，则先触发前置验证，过滤器链验证，再后置验证，共3个步骤

    4. 重点，前置方法中调用访问决策管理器验证是否可以访问

      1. 参数一：认证对象，上下文对象中取出，判断是否认证即isAuthenticated()，没有则用认证管理器认证
      2. 参数二： 过滤器链对象，包括请求对象等信息
      3. 参数三：权限对象集合，由安全元数据对象的getAttributes方法获取，默认取WebSecurity加载的配置

    5. 访问决策管理器默认实现AffirmativeBased，采用投票决策，根据结果判断是否报异常。

    # 开发参考

    通过以上主要步骤得出，实际开发的几种实现方式：

    • 自定义访问决策对象，实现AccessDecisionManager接口

    • 重写安全元数据对象，即实现FilterInvocationSecurityMetadataSource接口，重写getAttributes()

    • 一般以上两种一起用，或者直接用默认实现，直接配置元数据

    • 自定义过滤器链，参考源码改

        //http.addFilterAt(MyFilterSecurityInterceptor.init(), FilterSecurityInterceptor.class);
        
        /** 其他配置 */
    }
}

    表达式 说明
    permitAll 永远返回true
    denyAll 永远返回false
    anonymous 当前用户是anonymous时返回true
    rememberMe 当前用户是rememberMe用户时返回true
    authenticated 当前用户不是anonymous时返回true
    fullAuthenticated 当前用户既不是anonymous也不是rememberMe用户时返回true
    hasRole（role） 用户拥有指定的角色权限时返回true
    hasAnyRole（[role1，role2]） 用户拥有任意一个指定的角色权限时返回true
    hasAuthority（authority） 用户拥有指定的权限时返回true
    hasAnyAuthority（[authority1,authority2]） 用户拥有任意一个指定的权限时返回true
    hasIpAddress（'192.168.1.0'） 请求发送的Ip匹配时返回true

    # 其他过滤器

    上下文对象过滤器

    • 过滤器：SecurityContextPersistenceFilter

    • 上下文对象：SecurityContext

    • 作用：根据request获取HttpSession，再从HttpSession中获取上文下对象，每次请求后respose中带回Cookie，所以下次请求的Cookie中包含HttpSession信息

    • ps：原生security就是基于session来解析认证对象，来保证后续认证鉴权

    匿名对象过滤器

    • 过滤器：AnonymousAuthenticationFilter

    • 作用：上下文对象中不存在认证对象，则封装一个AnonymousAuthenticationToken认证对象

    jwt的扩展，其中一步就是自定义jwtToken的解析过滤器，请求头获取、解析，验证，最终存到上下文对象中

    详情，后续的spring cloud oauht2再分析（OAuth2AuthenticationProcessingFilter）

    # 异常处理

    访问异常处理分为以下几种：

    • 未登录访问：异常AuthenticationException，实现AuthenticationEntryPoint接口，重写commence()，默认实现LoginUrlAuthenticationEntryPoint跳转登录页
    • 已登录，但无权访问：异常AccessDeniedException，实现AccessDeniedHandler接口，重写handle()，默认实现AccessDeniedHandlerImpl返回403

    登录、登出处理分为以下几种：

    • 登录认证成功：实现AuthenticationSuccessHandler接口，重写onAuthenticationSuccess()，默认实现SavedRequestAwareAuthenticationSuccessHandler以及抽象父类
    • 登录认证失败：实现AuthenticationFailureHandler接口，重写onAuthenticationFailure()，默认实现SimpleUrlAuthenticationFailureHandler以及抽象父类
    • 成功登出处理：实现LogoutSuccessHandler接口， 重写onLogoutSuccess()，默认实现SimpleUrlLogoutSuccessHandler

    并发Session

    • 基于SessionManagementFilter过滤器进行session认证，由SessionAuthenticationStrategy管理认证。
    • 参考 (opens new window)
    • jwt认证不考虑，session、cookie认证需要考虑

    # 安全配置

    @EnableConfigurationProperties({YHSecurityProperties.class})
@ConditionalOnProperty(
        prefix = "yh.security",
        value = "enabled",
        matchIfMissing = true
)
//@EnableWebSecurity
public class YHWebSecurityAutoConfiguration extends WebSecurityConfigurerAdapter {
    @Autowired
    private DemoAuthenticationProvider demoAuthenticationProvider;
    @Autowired
    private MyUserDetailsManager myUserDetailsManager;
    @Autowired
    private MyAccessDeniedHandler myAccessDeniedHandler;
    @Autowired
    private MyLoginUrlAuthenticationEntryPoint macLoginUrlAuthenticationEntryPoint;
    
    /** 配置读取 */
    @Autowired
    private  YHSecurityProperties yhSecurityProperties;

    @Override
    public void init(WebSecurity web) throws Exception {
        //默认加载FilterSecurityInterceptor，点击父类方法查看
        super.init(web);
    }

    @Override
    protected void configure(AuthenticationManagerBuilder auth) throws Exception {
        //在内存中创建一个名为 "user" 的用户，密码为 "pwd"，拥有 "USER" 权限，密码使用自定义实现类CustomPwdEncoder加密
        auth.inMemoryAuthentication().passwordEncoder(new CustomPwdEncoder())
                .withUser("user").password(new CustomPwdEncoder().encode("pwd")).roles("USER");
        //将自定义验证类注册进去
        auth.authenticationProvider(demoAuthenticationProvider);
        //加入数据库验证类，下面的语句实际上在验证链中加入了一个DaoAuthenticationProvider
        auth.userDetailsService(myUserDetailsManager).passwordEncoder(new BCryptPasswordEncoder());
    }

    @Override
    public void configure(WebSecurity web) throws Exception {
        //security 直接忽略的请求，一般配置静态文件
        web.ignoring()
                //框架默认的一些请求
                .antMatchers("/error", "/csrf", "/*.ico")
                //swagger页面的请求
                .antMatchers("/swagger-ui.html", "/swagger-resources/**", "/v2/api-docs/**", "/webjars/**")
                //前端测试页面请求，本项目的测试模块在view包
                .antMatchers("/", "/index", "/login_y");
    }

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        /** 权限访问配置，默认FilterSecurityInterceptor */
        http.authorizeRequests()
                .antMatchers("/depart2").hasAnyRole("ROLE_DEPART2")
                .anyRequest().authenticated();
        /** 新增自己的过滤器链，默认FilterSecurityInterceptor排序为最后个 */
        //http.addFilterAt(MyFilterSecurityInterceptor.init(), FilterSecurityInterceptor.class);
        
        /** 新增其他过滤器 */
        http.addFilterBefore(RefererCsrfFilter.init(yhSecurityProperties), CsrfFilter.class);
        
        /** 登录页配置 */
        http.formLogin()
                //自己的登录页面
                .loginPage("/login_h")
                .loginProcessingUrl("/login")
                .usernameParameter("myUserName").passwordParameter("myPassword")
                //以上登录地址也属于可访问
                .permitAll();
        /** 登出页配置 */
        http.logout()
                .logoutSuccessUrl("/login_h")
                //适合前后端分离配置
//                .logoutSuccessHandler(new MyLogoutSuccessHandler())
                .permitAll();
        /** 异常处理 */
        http.exceptionHandling()
                //未登录无权访问异常处理
                .authenticationEntryPoint(macLoginUrlAuthenticationEntryPoint)
                //已登录无权访问异常处理
                .accessDeniedHandler(myAccessDeniedHandler)
                //无效访问跳转的页面
//                .accessDeniedPage("")
                ;
        /** 其他配置 */
        http.csrf().disable();
    }
}

          public static Authentication login(HttpServletRequest request, String userName, String pwd, boolean isIgnorePwd) {
        AuthenticationManager authenticationManager = MySpringUtils.getBean("authenticationManager");
        CustomPwdEncoder.setIgnore(isIgnorePwd);
        UsernamePasswordAuthenticationToken authRequest = new UsernamePasswordAuthenticationToken(userName, pwd);
        authRequest.setDetails(new WebAuthenticationDetails(request));
        SecurityContext securityContext = SecurityContextHolder.getContext();
        Authentication auth = authenticationManager.authenticate(authRequest);
        securityContext.setAuthentication(auth);
        return auth;
    }
