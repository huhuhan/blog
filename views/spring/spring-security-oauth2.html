          Spring Security Oauth2 总结

          寒铁 2021-01-21 spring source

          前提是了解Spring Security

          # 简介

          即引入依赖包，默认加载的AuthorizationServerEndpointsConfiguration

                  <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-starter-oauth2</artifactId>
        </dependency>
          1
          2
          3
          4

          该类会初始化注入以下的Endpoint类，包括以下：

          • AuthorizationEndpoint：授权，对应请求/oauth/authorize
          • TokenEndpoint：认证，对应请求/oauth/token
          • CheckTokenEndpoint，对应请求/oauth/error
          • WhitelabelApprovalEndpoint，对应请求/oauth/confirm_access
          • WhitelabelErrorEndpoint，对应请求/oauth/error

          相对Spring Security的WebSecurityConfigurerAdapter，oauth2有两种安全配置

          • 授权服务配置AuthorizationServerConfigurerAdapter
          • 资源服务配置ResourceServerConfigurerAdapter
            • 默认注解EnableResourceServer的配置也是继承WebSecurityConfigurerAdapter实现的
            • WebFlux版的比如Spring Cloud Gateway，则重定义SecurityWebFilterChain

          # 授权服务配置

          这个系统暂称为【授权中心】，自定义授权配置类，继承AuthorizationServerConfigurerAdapter类，通过实现以下三种configure方法来配置。最简单的配置，如下

          # 访问安全配置

          框架EndpointRequest的访问安全配置。其他请求的访问安全按Spring Security的安全配置。

              @Override
    public void configure(AuthorizationServerSecurityConfigurer security) throws Exception {
        security
                //设置/oauth/check_token可以无权访问，给资源服务远程访问用
                .checkTokenAccess("permitAll()")
            	//可选，默认client_id以及client_secret要求以basic auth加密形式请求
                .allowFormAuthenticationForClients();
    }

          # 客户端配置

          所有客户端或应用需要先注册到【授权中心】中，客户端根据密钥来【授权中心】获取授权令牌

          	@Override
    public void configure(ClientDetailsServiceConfigurer clients) throws Exception {
        //从内存中加载客户端信息
        clients.inMemory()
                //客户端ID，必填
                .withClient("client_demo")
                //客户端秘钥, security5.0后要求秘钥加密
                .secret(passwordEncode.encode("secret_client_demo"))
                //作用域
                .scopes("all")
                //支持的授权模式，最简化配置仅客户端模式可用
                .authorizedGrantTypes(AuthorizationGrantType.CLIENT_CREDENTIALS.getValue())
                //授权客户端的权限
                .authorities("client_x")
                //访问token时效，秒
                .accessTokenValiditySeconds(3600)
        ;
    }

          # 开启授权码模式

          类似QQ授权给第三方应用。访问跳转至QQ，检查是否已登录，是否授权，确认授权，跳转回第三方应用地址拿到授权码或令牌。这里QQ就是【授权中心】，客户端服务即第三方应用。

                        AuthorizationGrantType.AUTHORIZATION_CODE.getValue(),
                        AuthorizationGrantType.CLIENT_CREDENTIALS.getValue()

                //刷新token时效，秒
                .refreshTokenValiditySeconds(3600)
                //访问token时效，秒
                .accessTokenValiditySeconds(3600)

          # 重写异常响应

          在ResourceServerConfigurerAdapter中配置，主要涉及以下几个接口

          1. AuthenticationEntryPoint: 无身份认证对象访问，401需要身份认证
2. AccessDeniedHandler: 已登录无权访问，403正确请求但拒绝请求
3. LogoutSuccessHandler: 成功退出登录后拦截处理
4. LogoutHandler: 退出登录，拦截处理
          1
          2
          3
          4
              @Override
    public void configure(ResourceServerSecurityConfigurer resources) {
        resources.stateless(true)
                .authenticationEntryPoint(authenticationEntryPoint)
                .expressionHandler(expressionHandler)
                .accessDeniedHandler(accessDeniedHandler);
    }

    @Override
    public HttpSecurity configure(HttpSecurity http) throws Exception {
        // 退出处理
        http.logout()
                .addLogoutHandler(logoutHandler)
                .logoutSuccessHandler(logoutSuccessHandler)
                // 清除上下文对象中的认证对象
                .clearAuthentication(true);
        return http;
    }

          # 令牌扩充优化

          即扩展响应对象。可以参考源码的请求/oauth/token类TokenEndpoint，响应对象OAuth2AccessToken。主要接口TokenEnhancer就是将数据扩充到响应对象OAuth2AccessToken的additionalInformation属性中。

          下面两种方式都是基于DefaultTokenServices生成token、refreshToken的

          # 自定义扩充

          • 自定义实现TokenEnhancer接口

            	@Bean
    public TokenEnhancer tokenEnhancer() {
        return new TokenEnhancer() {
            @Override
            public OAuth2AccessToken enhance(OAuth2AccessToken accessToken, OAuth2Authentication authentication) {
                Authentication userAuthentication = authentication.getUserAuthentication();
                if (userAuthentication != null) {
                    String userName = userAuthentication.getName();
                    //数据库查询用户，加到token中
                    if (null != userName || true) {
                        Map<String, Object> additionalInformation = new HashMap<>();
                        Map<String, Object> map = new HashMap<>();
                        map.put("userName", "admin");
                        map.put("createTime", new Date().toString());
                        map.put("gender", "男");
                        additionalInformation.put("roles", Arrays.asList("admin", "person"));
                        additionalInformation.put("user", map);//GsonUtil.toJson(map));
                        ((DefaultOAuth2AccessToken) accessToken).setAdditionalInformation(additionalInformation);
                    }
                }
                return accessToken;
            }
        };
    }

          # 令牌存储优化

          主要通过接口TokenStore实现，默认实现类是InMemoryTokenStore，即存储在内存中。

          上面的例子都是默认内存存储，【资源服务】的验证需要远程请求去【授权中心】的TokenStore验证。

          只要保证创建的token和验证的token来自同个TokenStore，就没问题！

          依赖包中默认提供了几种实现类

          # 授权码存储优化

          仅限于授权模式。生成的授权码存储接口AuthorizationCodeServices

          • 默认实现：InMemoryAuthorizationCodeServices，只要请求获取token成功才删除内存中的随机码。
          • 数据库：JdbcAuthorizationCodeServices
          • 扩展如Redis实现方式，设置随机码的有效期，继承抽象类RandomValueAuthorizationCodeServices

    /**
     * 自定义令牌授权器，扩展授权模式
     *
     * @return
     */
    @Bean
    public TokenGranter tokenGranter() {
        return new TokenGranter() {
            private CompositeTokenGranter delegate;

            @Override
            public OAuth2AccessToken grant(String grantType, TokenRequest tokenRequest) {
                if (this.delegate == null) {
                    this.delegate = new CompositeTokenGranter(getDefaultTokenGranters());
                }
                return this.delegate.grant(grantType, tokenRequest);
            }
        };
    }

    /**
     * 源码方法，扩展授权模式
     *
     * @return
     */
    private List<TokenGranter> getDefaultTokenGranters() {
        ClientDetailsService clientDetails = clientDetailsService;
        AuthorizationServerTokenServices tokenServices = this.tokenServices();
        AuthorizationCodeServices authorizationCodeServices = this.authorizationCodeServices();
        OAuth2RequestFactory requestFactory = this.requestFactory();
        List<TokenGranter> tokenGranters = new ArrayList<>();
        // 授权码模式 authorization_code
        tokenGranters.add(new AuthorizationCodeTokenGranter(tokenServices, authorizationCodeServices, clientDetails, requestFactory));
        // 刷新令牌模式 refresh_token
        tokenGranters.add(new RefreshTokenGranter(tokenServices, clientDetails, requestFactory));
        // 隐式模式 implicit
        tokenGranters.add(new ImplicitTokenGranter(tokenServices, clientDetails, requestFactory));
        // 客户端模式 client_credentials
        tokenGranters.add(new ClientCredentialsTokenGranter(tokenServices, clientDetails, requestFactory));
        if (this.authenticationManager != null) {
            // 用户密码模式 password
            tokenGranters.add(new ResourceOwnerPasswordTokenGranter(this.authenticationManager, tokenServices, clientDetails, requestFactory));
        }
        // todo： 扩展补充其他授权码方法，比如验证码、手机号、第三方应用等等
        return tokenGranters;
