# 简介

Spring Cloud 微服务框架中常用的一套组件，现已停止维护更新，不影响使用。

  • 组件介绍：服务发现（Eureka），断路器（Hystrix），网关路由（Zuul）、客户端负载平衡（Ribbon）、统一网关（Zuul）
  • 其他组件：
    • 服务调用，Feign异步RPC框架，集成HTTP请求、配合Ribbon、Hystrix使用
    • 统一网关，Gateway，Zuul的替代组件
  • 推荐网站：Spring Cloud 中文网 (opens new window)
  • 简单测试项目：demo-sc (opens new window)

# Feign

HTTP URL 请求客户端类型：

  • HttpURLConnection: 默认，jdk自带，无连接池，参考源码feign.Client.Default
  • ApacheHttpClient: 自动化配置默认开启，单独引入httpclient依赖包，有连接池
  • OkHttpClient: 自动化配置默认关闭，单独引入okhttp依赖包，有连接池
  • 集成Ribbon：先根据Ribbon负载均衡选择服务，再以上三种客户端之一调用

自动化配置：参考源码FeignAutoConfiguration和HttpClientFeignLoadBalancedConfiguration

  • HTTP 请求的日志级别，打印通信请求的日志，分4级（NONE、BASIC、HEADERS、FULL）
  • 支持GIZ压缩设置：请求、响应的数据压缩，加快通信
  • 支持编码解码器设置：转化数据格式，比如 Gson、Jaxb、Jackson
  • 支持熔断器配置：默认关闭，和Hystrix依赖包一起用
  • 支持Ribbon负载均衡：默认集成

# Feign-转发拦截器

场景：网络隔离架构下，仅统一网关开放，即认证鉴权集成在网关中，其他服务无安全拦截。请求认证，比如token、tenantId等信息需要传递其他服务，一般存储在请求头中。

  • 如某请求，网关G >> 服务A >> 服务B，服务B中没有网关G解析的请求头参数

  • hystrix的默认隔离策略是线程池，即THREAD

  • 服务A转发服务B，转发请求头参数需要设置，但隔离线程池中的线程获取不到原始的ServletRequestAttributes对象

  • 服务A的配置修改如下：

    • 隔离策略，设置为信号量，即SEMAPHORE

    • 添加拦截器，实现RequestInterceptor接口，转发前拦截，重新补充请求头参数

# Hystrix隔离策略

场景：服务A设置最大可用线程1000，某时刻并发请求500，请求调用服务B，服务B某原因挂掉，导致线程阻塞，逐渐的，服务A也因为线程不够用而挂掉，最终导致整个系统的服务挂了

  • 以上情况，即服务雪崩效应。服务A配置隔离策略，服务A超过设置的并发量就走降级处理。

  • 信号量（SEMAPHORE）

    • 每个线程（用户）请求获取一个信号，处理结束，归还信号，计数信号量。
    • 通过计数，达到最大数后，再加则触发fallback降级处理
    • 同步调用（用户请求和业务执行HystrixCommand 是同一线程）
    • 适合隔离响应速度快的服务，不影响线程阻塞，比如redis

  • 线程（THREAD），默认配置

    • 每个线程（用户）请求，发送到Hystrix的线程池中，单独起一个线程处理。
    • 业务执行HystrixCommand，单独的线程，超时则触发fallback熔断
    • 异步调用（同步也行），根据实现应用而定
    • 线程池发起的线程超时，则触发fallback熔断
    • 线程上下文切换，相对消耗资源大点

# Eureka

从功能分类来看：

  • 服务注册：Eureka Client 启动时发送请求注册到Eureka。
  • 服务续约：Eureka Client 启动时的定时任务，周期性发送请求，Eureka更新服务信息。（心跳机制）
  • 服务下线：Eureka Client 关闭服务，发送请求到Eureka，Eureka删除服务信息 或者 Eureka剔除下线
  • 服务获取：Eureka Client 启动后的定时任务，周期性发送请，获取Eureka中的服务清单，缓存本地用于服务调用。
  • 服务同步：Eureka 部署集群时，在服务注册、服务续约、定时任务情况下都会同步更新至其他Eureka节点

从角色分类来看：

  • 服务消费者、服务提供者：即Eureka Client服务，调用其他服务接口和提供其他服务接口，一般同时兼任
  • 服务注册中心：即Eureka Server服务，同时集群部署，实现高可用

# 心跳机制

  • Eureka Client注册时，发送请求到Eureka，注册成功，服务信息保存在Eureka。
  • 后续，Eureka Client 定期发送心跳请求到Eureka，更新服务信息
    • 心跳请求，默认30秒发一次。（服务续约周期）
    • 续约过期时长，默认90秒。（服务续约过期时长）
  • Eureka 定期检测，剔除无效不健康的服务实例，周期默认60秒。（服务剔除周期）
    • 无效服务的过期检测依据：当前时间 > 服务最后心跳时间 + 续约过期时长

源码参考：Eureka的InstanceResource类的请求接口

# 自我保护机制

  • Eureka 可能因为网络问题没有收到服务的心跳请求，导致定期检测的无效服务实例，实际并非无效。
  • 为了尽量避免这种情况，触发保护机制：默认定期每15分钟检测，超过85%的服务都没收到心跳。（自我保护系数，默认0.85）
    • 分配删除无效服务，比如20个实例，5个失效，先计算最大允许实例=实例*自我保护系数
    • 第一轮: 允许实例20*0.85=17，应删除20-17=3个，剩余5-3=2个
    • 第二轮：允许实例17*0.85=14，应删除17-14=3个，剩余0个
    • 因此，服务数量多的情况，考虑减小自我保护系数
  • 触发后：
    • 不会再注销因心跳机制验证的无效服务
    • 可以接收新服务注册和查询服务的请求
    • 暂停集群节点信息的同步，等网络确认稳定，再恢复同步

# 负载均衡机制

  • 采用Ribbon的负载均衡，默认规则时轮询
  • Eureka Client通过定期（默认30秒）的服务获取，将注册中心的可用服务实例清单缓存在本地，用于接口调用

# Zuul

功能：

  • 动态路由
  • 过滤器拦截：基于ZuulFilter接口（pre、route、post、error共四种过滤类型）
  • 重试机制：基于Ribbon的重试机制，切换转发不同服务
  • 回退机制：基于Hystrix的熔断降级

# Ribbon重试机制

Eureka基于AP原则构建，缺失一致性。当心跳检查机制无法同步时，Ribbon获取到的错误服务信息，会发生失败请求，通过重试机制来避免。

  • 重试机制，即次数不包括原本请求
  • 开启OkToRetryOnAllOperations对所有请求方式触发，如POST、PUT。可能造成业务数据重复操作现象
  • 重试次数 = MaxAutoRetries + MaxAutoRetriesNextServer + (MaxAutoRetries * MaxAutoRetriesNextServer)
  • 默认重试次数 = 0 + 0 + ( 0 *0 ) = 0次
  • 最终结果还是超时，则总超时时长 = ReadTimeout * ( 1 + 重试次数 )

# Eureka-服务端

# 服务注册中心
eureka:
  instance:
    hostname: hs-eureka
  client:
    # 服务注册，默认开启，Eureka自身服务不需要注册
    register-with-eureka: false
    # 服务获取，默认开启，Eureka自身服务不作为消费者
    fetch-registry: false
    service-url:
      # 若搭建集群，${server.port}改为其他节点的端口，互相注册，多个服务配置通过逗号间隔；这里集成security的认证
      defaultZone: http://yh:yh@${eureka.instance.hostname}:${server.port}/eureka/
  server:
    # 自我保护机制，默认开启，适合开发阶段关闭
    enable-self-preservation: true
    # 自我保护机制，自我保护系数，默认0.85 
    renewal-percent-threshold: 0.85
    # 心跳机制，服务剔除周期，单位毫秒，默认60秒，[AbstractInstanceRegistry]类的定时器会打印日志
    eviction-interval-timer-in-ms: 60000
    # 获取不到实例时等待时间，单位毫秒，默认300秒
    waitTimeInMsWhenSyncEmpty: 300000
    # 关闭REST API 响应缓存
    use-read-only-response-cache: false

# Feign

优先级：细粒度属性配置 > 细粒度代码配置 > 全局属性配置 > 全局代码配置

feign:
  # 默认JDK的HttpURLConnection请求
  client:
    # 自由配置，IDE没有提示，和hystrix.command配置类似，
    config:
      # 全局配置，适用于所有FeignClient类
      default:
        loggerLevel: FULL
      # 指定某服务的@FeignClient，取contextId，默认等于name值
      contextId:
        loggerLevel: FULL     # 日志级别，[feign.Logger.Level]
        connectTimeout: 5000  # 相当于Request.Options 连接超时时间
        readTimeout: 5000     # 相当于Request.Options 读取超时时间
        errorDecoder: com.example.SimpleErrorDecoder  # Feign的错误解码器，相当于代码配置方式中的ErrorDecoder
        retryer: com.example.SimpleRetryer  # 配置重试，相当于代码配置方式中的Retryer
        requestInterceptors:  # 配置拦截器，相当于代码配置方式中的RequestInterceptor
          - com.example.FooRequestInterceptor
          - com.example.BarRequestInterceptor
        decode404: false      # 是否对404错误解码
        encode: com.example.SimpleEncoder
        decoder: com.example.SimpleDecoder
        contract: com.example.SimpleContract
  # 采用httpClient请求
  httpclient:
    # 默认开启，需要单独引入feign-httpclient依赖包
    enabled: true
    max-connections: 200
    connection-timer-repeat: 3000
    # 其他参考源码[FeignHttpClientProperties]
  # 采用okhttp请求，不支持GZip压缩
  okhttp:
    # 默认关闭，开启并单独引入feign-okhttp依赖包
    enabled: false
  # GIZ 压缩
  compression:
    request:
      # 默认关闭，开启增加CPU压力，但有效节省网络资源
      enabled: false
      # 默认最小请求参数的长度，开启调大这个参数
      min-request-size: 2048
      # 默认请求类型
      mime-types: text/xml, application/xml, application/json
    response:
      enabled: false
      useGzipDecoder: false

# Ribbon

# 重试次数 = MaxAutoRetries + MaxAutoRetriesNextServer + (MaxAutoRetries * MaxAutoRetriesNextServer)

  # 最大请求重试次数，负债均衡切换不同实例，默认0
  MaxAutoRetriesNextServer: 0

# Hystrix

优先级：细粒度属性配置 > 细粒度代码配置 > 全局属性配置 > 全局代码配置

feign:
  hystrix:
    # 默认禁用hystrix，需要开启
    enabled: true
hystrix:
  # 默认隔离策略，线程池配置
  threadpool:
    default:
      # 最大线程数，默认10
      coreSize: 10
      # 最大队列数，默认值-1，线程满了进入阻塞队列等待， 默认-1
      maxQueueSize: -1 
      # 最大队列数不可热修改，该值控制最大队列数，达到该值请求也拒绝，默认值5
      queueSizeRejectionThreshold: 5
      # 线程存活时间，默认1分钟 
      keepAliveTimeMinutes: 1
  # 自由配置，IDE没有提示，和feign.client配置类似
  command:
    # 全局有效
    default:
      execution:
        timeout:
          # 默认true，根据超时配置做熔断，为false，超时判断交给ribbon来熔断，报错异常不同
          enabled: false
        isolation:
          # 隔离策略，默认，THREAD，即线程池； 可选，SEMAPHORE，即信号量
          strategy: THREAD
          semaphore:
            # 最大信号量，超过就触发降级，与超时配置无关
            maxConcurrentRequests: 10
          thread:
            # 超时时间，默认1000ms，可设置为ribbon总超时时长
            timeoutInMilliseconds: 1000
            # 默认值为 true，执行超时则中断，中断后降级处理
            interruptOnTimeout: true
    # 自定义熔断器，FeignClient引入配置command
    MyFallbackcmd:
      execution:
        isolation:
          thread:
            timeoutInMilliseconds: 9000
    # 指定某个FeignClient的某个方法
    'HelloFeignClient#testHystrix':
      execution:
        isolation:
          thread:
            timeoutInMilliseconds: 9000
