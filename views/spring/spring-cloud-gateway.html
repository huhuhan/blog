      Webflux | Spring Cloud Gateway

      寒铁 2020-11-27 spring

      # WebFlux

      • Spring Webflux中的reactor-netty响应式编程组件，

        • 异步非阻塞式，基于Reactor库的异步和事件驱动，不代表缩短请求时间

        • 函数式编程、吞吐量高、伸缩性强

        • 请求分发类DispatcherHandler

        • 请求和响应：ServerHttpRequest和ServerHttpResponse，信息封装对象ServerWebExchange

        • 适合IO密集型服务，如网关服务（转发的吞吐量高）

          IO密集型包括：磁盘IO密集型, 网络IO密集型，网关是网络IO密集型

        • 默认以Netty为服务器

        • 不支持MySql、不方便Debug调试

      • 相对于Spring MVC

        • 基于Servlet API，使用同步阻塞式 I/O 模型，即一个请求一个线程

          Servlet 3.1也是异步非阻塞式模型

        • 请求分发类DispatcherServlet

        • 默认以Tomcat为服务器（spring-boot-web依赖包）

      # Gateway

      • 相对Zuul

        • SpringCloud2.0前是Zuul，基于Servlet IO处理模型，请求分发类DispatcherServlet
        • Zuul2.0后续版本基于Servlet3.1，也是异步非阻塞，但SpringCloud后续版本没有集成

      • 认证鉴权

        • 集成是 spring security reactive，也是基于spring webflux框架

      # 动态路由

      • 自动化配置加载类：GatewayAutoConfiguration
      • 一组route必填id、uri、predicates，可选filters，源码对象RouteDefinition
      • 路由对象存储接口RouteDefinitionRepository
        • 默认实现InMemoryRouteDefinitionRepository，即存储在内存中，可自定义扩展实现

      # URI

      • 支持http、https地址，如http://www.baidu.com
      • 服务发现的地址：格式lb://my-demo-client
      • 多级转发，最后加两个*：比如- Path=/sever-user/**

      # Predicates

      • 源码包目录：org.springframework.cloud.gateway.handler.predicate
      spring:
  cloud:
    gateway:
      # 自定义路由转发规则
      routes:
        # 一组规则，id唯一
        - id: after_route
          uri: http://www.baidu.com 
          predicates:
            ## 源码：AfterRoutePredicateFactory
            ## 在这时间后，请求都会转发
            - After=2018-12-25T14:33:47.789+08:00
        - id: before_route
          uri: http://www.baidu.com
          predicates:
            ## 源码：BeforeRoutePredicateFactory
            ## 在这时间前请求，都会转发
            - Before=2018-12-25T14:33:47.789+08:00
        - id: between_route
          uri: http://www.baidu.com
          predicates:
            ## 源码：BetweenRoutePredicateFactory
            ## 在这2个时间之间，请求都会转发
            - Between=2018-12-25T14:33:47.789+08:00, 2018-12-26T14:33:47.789+08:00
        - id: cookie_route
          uri: http://www.baidu.com
          predicates:
            ## 源码：CookieRoutePredicateFactory
            ## 匹配key=cookiename和value=cookievalue，就转发
            - Cookie=cookiename, cookievalue
        - id: header_route
          uri: http://www.baidu.com
          predicates:
            ## 源码：HeaderRoutePredicateFactory
            ## 匹配，头名称=X-Request-Id，值=数字，就转发
            - Header=X-Request-Id, \d+
        - id: host_route
          uri: http://www.baidu.com
          predicates:
            ## 源码：HostRoutePredicateFactory
            ## 匹配，以下域名，就转发
            - Host=**.somehost.org,**.anotherhost.org
        - id: method_route
          uri: http://www.baidu.com
          predicates:
            ## 源码：MethodRoutePredicateFactory
            ## 匹配，GET请求方式，就转发
            - Method=GET
        - id: path_route
          uri: http://www.baidu.com
          predicates:
            ## 源码：PathRoutePredicateFactory
            ## 匹配，/foo/**，/bar/**，segment为变量，就转发
            - Path=/foo/{segment},/bar/{segment}
        - id: query_route
          uri: http://www.baidu.com
          predicates:
            ## 源码：QueryRoutePredicateFactory
            ## 匹配，查询参数username，且正则匹配，参数值为zhang开头的，就转发
            - Query=username，zhang
        - id: remoteaddr_route
          uri: http://www.baidu.com
          predicates:
            ## 源码：RemoteAddrRoutePredicateFactory
            ## 匹配，ip范围在1~24的，就转发
            - RemoteAddr=192.168.0.1/24

      # Filters

      • 源码目录：org.springframework.cloud.gateway.filter.factory
      • 以下配置中省略必填项predicates
      spring:
  cloud:
    gateway:
      routes:
        - id: add_request_header_route_1
          uri: http://www.baidu.com
          filters:
            ## 源码：AddRequestHeaderGatewayFilterFactory
            ## 过滤，添加请求头参数，X-Request-Foo:Bar，再转发
            - AddRequestHeader=X-Request-Foo, Bar

        - id: add_request_parameter_route_2
          uri: http://www.baidu.com
          filters:
            ## 源码：AddRequestParameterGatewayFilterFactory
            ## 过滤，添加查询参数，参数foo值bar，再转发
            - AddRequestParameter=foo, bar

        - id: add_response_header_route_3
          uri: http://www.baidu.com
          filters:
            ## 源码：AddResponseHeaderGatewayFilterFactory
            ## 过滤，添加响应头参数，X-Response-Foo:Bar，再转发
            - AddResponseHeader=X-Response-Foo, Bar

        - id: preserve_host_route_4
          uri: http://www.baidu.com
          filters:
            ## 源码：PreserveHostHeaderGatewayFilterFactory
            ## 过滤，保留原始请求的host头信息，再转发
            - PreserveHostHeader

        - id: prefixpath_route_5
          uri: http://www.baidu.com
          filters:
            ## 源码：PrefixPathGatewayFilterFactory
            ## 过滤，路由重写，再转发，即/get转发为/my/get
            - PrefixPath=/my

        - id: redirectTo_route_6
          uri: http://www.baidu.com
          filters: 	
            ## 源码：RedirectToGatewayFilterFactory
            ## 过滤，根据状态码（300系列），转发，即直接转发百度
            - RedirectTo=302, http://www.baidu.com

        - id: rewritepath_route_7
          uri: http://www.baidu.com
          filters:
            ## 源码：RewritePathGatewayFilterFactory
            ## 过滤，重写路由，再转发，即/foo/bar转发为/bar
            - RewritePath=/foo/(?<segment>.*), /$\{segment}
            
        - id: stripprefix_route_8
          uri: http://www.baidu.com
          filters:
            ## 源码：StripPrefixGatewayFilterFactory
            ## 过滤，截断路由，再转发，即/a/b/bar转发为/bar
            - StripPrefix=2
            
        - id: requestsize_route_9
          uri: http://www.baidu.com
          filters:
            ## 源码：RequestSizeGatewayFilterFactory
            ## 过滤，限制请求包大小，再转发
	        - name: RequestSize
      			args:
          		maxSize: 5000000 # 默认5MB

      • 降级处理接口：/my-fallback

      # 熔断超时时间，单位：毫秒，默认2秒
# 请求的服务超过设置的时间没有响应则熔断
# 源码：HystrixCommandProperties.executionTimeoutInMilliseconds
hystrix:
  command:
    # 全局HystrixCommand
    default:
      execution:
        isolation:
          thread:
            timeoutInMilliseconds: 5000
    # 自定义HystrixCommand
    fallbackcmd:
      execution:
        isolation:
          thread:
            timeoutInMilliseconds: 8000
# 配置参考例子
spring:
  cloud:
    gateway:
      routes:
        - id: hystrix_route
          uri: lb://DEMO-SERVER
          predicates:
            - Path=//DEMO-SERVER/**
          filters:
            ## Hystrix作为过滤规则：Hystrix GatewayFilter Factory
            ## 过滤，断路器降级处理，转发到指定接口
            - name: Hystrix
              args:
                name: fallbackcmd
                fallbackUri: forward:/my-fallback

        - id: fallbackHeaders_route
          uri: http://www.google.com
          filters:
            ## FallbackHeaders作为过滤规则：FallbackHeaders GatewayFilter Factory
            ## 过滤，将降级异常放到请求头，再转发
            - name: FallbackHeaders
              args:
                # 默认请求头参数名如下，可修改
                executionExceptionTypeHeaderName: Execution-Exception-Type
                executionExceptionMessageHeaderName: Execution-Exception-Message
                rootCauseExceptionTypeHeaderName: Root-Cause-Exception-Type
                rootCauseExceptionMessageHeaderName: Root-Cause-Exception-Message

@Bean
KeyResolver userKeyResolver() {

/**

@Bean
public KeyResolver ipKeyResolver() {
  return exchange -> Mono.just(exchange.getRequest().getRemoteAddress().getHostName());
}
/**
 * API限流

@Bean
KeyResolver apiKeyResolver() {
  return exchange -> Mono.just(exchange.getRequest().getPath().value());

      # 过滤接口

      package com.cloud.yh.gateway;

import lombok.extern.slf4j.Slf4j;
import org.springframework.cloud.gateway.filter.GatewayFilterChain;
import org.springframework.cloud.gateway.filter.GlobalFilter;
import org.springframework.core.Ordered;
import org.springframework.stereotype.Component;
import org.springframework.web.server.ServerWebExchange;
import reactor.core.publisher.Mono;

/**
 * 网关请求拦截
 *
 * @author yanghan
 * @date 2019/6/20
 */
@Slf4j
@Component
public class MyGatewayFilter implements GlobalFilter, Ordered {

    /**
     * 网关拦截转发，主要方法
     * @author yanghan
     * @param exchange  即Predicate断言，设置规则
     * @param chain     即Filter过滤器，处理过滤
     * @return reactor.core.publisher.Mono<java.lang.Void>
     * @date 2020/11/26
     */
    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        log.info("网关拦截：" + exchange.getRequest().getURI());
        return chain.filter(exchange);
    }

    @Override
    public int getOrder() {
        return 0;
    }

}

      # 统一异常

      Spring基于Webflux的默认统一异常拦截，源码原理默认实现：基于抽象类AbstractErrorWebExceptionHandler的实现类DefaultErrorWebExceptionHandler，关键抽象方法如下

        protected RouterFunction<ServerResponse> getRoutingFunction(ErrorAttributes errorAttributes) {
      //默认优先解析为网页视图，类型为text/html，其次解析为application/json
      return RouterFunctions.route(this.acceptsTextHtml(), this::renderErrorView).andRoute(RequestPredicates.all(), this::renderErrorResponse);
  }
