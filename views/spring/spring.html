# 其他

# SpEL表达式

SpEL 字面量：

  • 整数：#{8}
  • 小数：#{8.8}
  • 科学计数法：#{1e4}
  • String：可以使用单引号或者双引号作为字符串的定界符号。
  • Boolean：#{true}

SpEL引用bean , 属性和方法：

  • 引用其他对象:#{car}
  • 引用其他对象的属性：#{car.brand}
  • 调用其它方法 , 还可以链式操作：#{car.toString()}
  • 调用静态方法静态属性：#{T(java.lang.Math).PI}

SpEL支持的运算符号：

  • 算术运算符：+，-，*，/，%，^(加号还可以用作字符串连接)
  • 比较运算符：< , > , == , >= , <= , lt , gt , eg , le , ge
  • 逻辑运算符：and , or , not ，|
  • if-else 运算符(类似三目运算符)：？:(temary), ?:(Elvis)

# 注解

# 元注解

所有注解基于这4个基础注解定义扩展

  • @Retention：声明注解对象的存在范围（枚举RetentionPolicy）

  • @Inherited：声明子类可以继承此注解

  • @Documented：声明注解能够被javadoc等识别

  • @Target：声明注解的作用范围（枚举ElementType）

    @Retention(RetentionPolicy.RUNTIME)//运行时可以反射获取注解对象
@Inherited//子类继承父类的此注解
@Documented//可被javadoc等识别
//声明注解范围: 字段、枚举常量，方法
@Target({ElementType.FIELD, ElementType.METHOD})
public @interface MyTestAnnotation {
    String name();
    //注解属性，默认为11
    String age() default "11";
}

# @PostConstruct

  • 类加载时，初始化一些数据
  • 适用非静态无参方法，返回值为void
  • 同理，类销毁时用的注解@PreDestroy
  • spring初始化加载资源的其他方式：
    • 接口CommandLineRunner：其实现方法run()是服务启动后立即执行
    • 接口InitializingBean: 其实现方法afterPropertiesSet()，类加载完后执行
  • 执行顺序：构造方法Constructor >> @Autowired >> @PostConstruct >> InitializingBean >> CommandLineRunner

# @Conditional

  • 适用于类、方法
  • 与注入的注解一起用，比如@Bean
  • 属性：定义一个实现Condition接口的类，实现matches方法，返回true/false
  • 相关注解：
    • @ConditionalOnBean：当SpringIoc容器内存在指定Bean的条件
    • @ConditionalOnClass：当SpringIoc容器内存在指定Class的条件
    • @ConditionalOnExpression：基于SpEL表达式作为判断条件
    • @ConditionalOnJava：基于JVM版本作为判断条件
    • @ConditionalOnJndi：在JNDI存在时查找指定的位置
    • @ConditionalOnMissingBean：当SpringIoc容器内不存在指定Bean的条件
    • @ConditionalOnMissingClass：当SpringIoc容器内不存在指定Class的条件
    • @ConditionalOnNotWebApplication：当前项目不是Web项目的条件
    • @ConditionalOnProperty：指定的属性是否有指定的值
    • @ConditionalOnResource：类路径是否有指定的值
    • @ConditionalOnSingleCandidate：当指定Bean在SpringIoc容器内只有一个，或者虽然有多个但是指定首选的Bean
    • @ConditionalOnWebApplication：当前项目是Web项目的条件
    // 例子，按需配置redisTemplate
   	@Bean
    @ConditionalOnMissingBean(RedisTemplate.class)
    public RedisTemplate<String, Object> redisTemplate(RedisConnectionFactory connectionFactory) {
        RedisTemplate<String, Object> redisTemplate = new RedisTemplate<>();
        redisTemplate.setConnectionFactory(connectionFactory);
        RedisSerializer<String> keySerializer = RedisSerializer.string();
        redisTemplate.setKeySerializer(keySerializer);
        redisTemplate.setHashKeySerializer(keySerializer);
        redisTemplate.afterPropertiesSet();
        return redisTemplate;
    }

# @Cacheable、@CachePut、@CacheEvict

  • 通过@EnableCaching开始内置缓存

  • 标记在类（表示类中所有方法）、方法，将方法结果缓存下来

  • 类对象内部调用，不会触发缓存功能，切面的代理机制问题

  • 注解接口：Cache、CacheManager。第三方依赖包，如Redis会扩展实现。

  • 注解属性说明

    // 例子
@Cacheable(value={"users"}, key="#user.id", condition="#user.id>0")
// 或者
@Cacheable(cacheNames="users", keyGenerator="myKeyGenerator", unless="#user.id==1")
public User find(User user) {
  System.out.println("find user by user " + user);
  return user;
}

    • cacheNames/value ：用来指定缓存组件的名字

    • key ：缓存数据时使用的 key，默认是使用方法参数的值。支持spEL 表达式，如图。

    • keyGenerator ：key 的生成器。 key 和 keyGenerator 二选一使用

    • cacheManager ：可以用来指定缓存管理器。从哪个缓存管理器里面获取缓存。

    • condition ：可以用来指定符合条件的情况下才缓存。

    • unless ：否定缓存。条件为 true ，方法的返回值就不会被缓存。支持spEL 表达式。

    • sync ：是否使用异步模式。默认是方法执行完，同步方式将结果存在缓存中。

  • 注解对比

    • @Cacheable ：查询缓存，没有则执行方法，缓存方法结果。适合查询操作
    • @CachePut：执行方法，缓存结果。适合更新操作
    • @CacheEvict：执行方法，删除缓存结果。适合删除操作

  • 原理：类似AOP，通过切面去调用缓存操作接口，切面是通过代理对象的引用去触发，内部调用（this的引用）导致proxy失效，即切面失效。

  • 相关注解对比说明

    • @Configuration：该注解将类可以看成配置文件，通常和其他配合使用。
    • @EnableAutoConfiguration：在程序启动时自动加载配置。

        value = "com.yhcloud.activiti5",
        includeFilters = {
                @Filter(type = FilterType.ANNOTATION, classes = {Controller.class})
        },
        useDefaultFilters = false
)

# 加载顺序

  • bootstrap.yml（bootstrap.properties）用来程序引导时执行，应用于更加早期配置信息读取，如可以使用来配置application.yml中使用到参数等
  • application.yml（application.properties) 应用程序特有配置信息，可以用来配置后续各个模块中需使用的公共参数等。
  • bootstrap.yml 先于 application.yml 加载

# 常见属性

属性原理可以在IDE打开spring boot项目，点击属性查看源码了解

server:
  # web端口
  port: 8866
#  port: ${server.port} # 取外置配置参数
#  #port: ${random.int[2000,8000]} # 取随机值
  servlet:
    # 项目根路径
    context-path: /
    session:
      # 默认session时长30分钟，不操作的情况
      timeout: 1m
      
spring:
  profiles:
    # 当前指定配置文件，如application-dev.yml
    active: dev
  application:
    # 服务名，忽略大小写
    name: xxxx-ssssssss
  http:
    multipart:
      # 最大支持文件大小
      max-file-size: 100Mb
      # 最大支持请求大小
      max-request-size: 100Mb

      location: D:/tmp/tomcat
  datasource:
    # 使用数据库连接池druid
    druid:
      url: jdbc:mysql://xxxxxxxxxxxxxx?useUnicode=true&characterEncoding=utf-8&&autoReconnect=true&failOverReadOnly=false&useSSL=true&zeroDateTimeBehavior=convertToNull
      username: xxxxxxxx
      password: xxxxxxxxxxxxxxxxxxxxx
      driver-class-name: com.mysql.jdbc.Driver
      pool-prepared-statements: true
      #配置初始化大小、最大、最小
      initial-size: 10
      max-active: 50
      min-idle: 10
      max-wait: 60000
      #检测间隔时间，关闭空闲连接
      time-between-eviction-runs-millis: 60000
      #连接在池中的生存时间
      min-evictable-idle-time-millis: 120000
      validation-query: SELECT 'x'
      test-while-idle: true
      test-on-borrow: false
      test-on-return: false
      # 配置监控统计拦截的filters, 参考(https://www.jianshu.com/p/fae8fc159270)
      filters: stat,wall
      #开启druid监控页面
      stat-view-servlet:
        enabled: true
        # IP白名单
        allow: 127.0.0.1
