            # [rm] 删除文件目录

            • r：递归删除，适合目录删除
            • f：强制删除，不询问，适合目录删除

            # [mv] 重名或移动

            • f：强制覆盖，不询问
            • b：若覆盖（已存在文件），会备份原文件，文件名后缀加~
            • n：不覆盖

            # [cp] 复制

            • r： 递归复制，适用目录
            • b：若覆盖（已存在文件），会备份原文件，文件名后缀加~
            • n：不覆盖
            • 强制覆盖命令**\cp**：如\cp a.log b.log

            # [ln] 创建超链接

            创建超链接，即另一个位置创建同步链接。默认是硬链接，比如删除需要同时删除

            • b： 删除，覆盖以前建立的链接
            • s： 软链接(符号链接)，仅快捷文件不占空间

            # [tar] 备份还原

            备份还原文件（打包非压缩，压缩只是其中一个参数接口调用）打包文件后缀为[.tar]

            • v：显示执行过程
            • f：执行压缩文件, 作最后个参数
            • 备份还原指令参数
              • c：创建备份文件，即打包
              • x：还原备份文件，即解包
              • t：列出备份包中的文件
            • 压缩解压指令参数
              • z：调用gzip指令处理备份文件，后缀一般为[.tar.gz]或[.tgz]，后缀可自定义，主要容易区分
              • j：调用bzip2指令处理备份文件，后缀一般为[.tar.bz]或[.tbz]
            • C：备份指定目录下，默认当前目录
            # -cvf 备份
tar -cvf my.tar *.log
# -tvf 查看
tar -tvf my.tar
# -xvf 还原
tar -xvf my.tar
# -cvf 备份到/myDir下
tar -cvf my.tar *.log -C /myDir
# 压缩解压，参数前添加相关压缩指令参数，如 -zcvf、-zxvf

# 排除文件目录--exclude=，写在最前面
## --exclude-vcs排除版本控制目录，如.git或.svn
tar -zcvf bpm-ui.tar.gz bpm-ui  --exclude-vcs --exclude=bpm-ui/.idea

            # [vim] 新建编辑

            • 启动后默认进入命令模式
            • i切换到输入模式，以输入字符
            • ESC退出当前模式，切换到命令模式

            • 底线命令模式下，q表示退出， w表示保存，通常一起用，wq
            • 常用命令：切换输入i、保存退出:wq、正常退出:q、强制退出不保存:q!

            # [chmod | chown] 管理权限

            管理文件、目录的权限。用户权限分三级：所有者、群组用户、其他用户

            • 读|写|执行: 数字权限值4|2|1可累加
            • 读|写|执行: 字母权限值r|w|x可累加
            • 三级用户： 字母对应值u|g|o|a=u+g+o
            • R：目录下文件、子目录递归赋予相同权限
            # 所有用户可读可写可执行
chomd 777 tmp.txt
chomd a=rwx tmp.txt
# 所有用户添加可执行权限，+添加-取消=唯一设定
chomd a+x tmp.txt
# 其他用户只可执行
chomd ug=rwx,o=x tmp.txt

            下面的进阶命令一般都会多个命令拼接一起用，逻辑运算符，适合多个命令、选项命令使用

            # [find] 搜索文件

            搜索文件、目录，返回路径，格式find [路径] -[选项] [搜索条件] -[执行命令]

            • [路径]不填默认当前，搜索结果为在当下路径的相对路径

            • -[执行命令]不填默认-print，即标准输出打印到控制台

            • [搜索条件]建议都加上双引号，特殊值容易识别为其他命令选项

            • 选项：

              • name：按文件名，iname忽略大小写，支持pattern正则形式，如 "*.log"

              • user：按所属用户，适合权限分配用户的系统，非只有用root账户的系统

              • size：按文件大小，参数n，单位c(1字节，默认)、w(2字节)、b(512字节)、k、M、G

                • n：等于n
                • -n： 小于n，n以内
                • +n：大于n，n以外

              • mtime：n*24小时被修改过数据的文件，n参考以上，类似命令ctime是被修改过状态的

              • mmin：n分钟被修改过数据的文件，n参考以上，类似命令cmin是被修改过状态的

              • mindepth：从第n级目录开始搜索

              • maxdepath：至多搜索到第n级，适合根目录下搜索

              • type：按类型，参数如下

                • f：文件
                • d：目录
                • l：链接文件，如/etc/init.d
                • b：块设备文件，如/dev目录下
                • c：字符设备文件，如/dev目录下
                • p：通道文件，如systemd命令集、docker程序等会有

              • 其他命令

                # -prune：排除，写在其他选项之前，一般和path一起用
# 查找当前目录下所有普通文件、文件大小必须大于500字节
# 但排除 test目录 和 opt目录，但，
find . -path ./test -prune -o -path ./opt -prune -o -tyep f -a -size +500

# -perm：按权限，必须熟悉权限编号
# 即所有者可读可写
find . -perm 600

            # [grep] 匹配字符

            搜索文件内的字符串，返回所在行，格式grep -[选项] [匹配条件] [搜索对象]

            • [匹配条件]：即字符串，支持pattern正则形式，建议加上双引号

            • [搜索对象]：

              • 单个文本：如某一文件(即可读取的文本内容)或某个搜索结果，返回[匹配条件]的所在行
              • 多个文件： 即有多个文本，返回格式文件路径:(行号:)所在行，路径为相对路径
              • 目录：不支持，提示这是个目录，需要加递归选项搜索

            • 选项

              • i：忽略大小写

              • n：加上行号

              • r：递归搜索，搜索对象为目录时

              • l：只返回匹配的文件路径，适合[搜索对象]为多个文件时

              • v：反向选取，即返回没有匹配的行

              • o：只返回匹配的字符串

            # [fdisk] 分区

            添加硬盘后，还需要系统分区、挂载才可使用

            # 查看分区，如上图，/dev/sda为系统盘，/dev/sdb是后续加的硬盘，分了一个区，还剩16G

# 分区操作
fdisk /dev/sdb
# 输入p，当前分区情况
# 输入d，删除某分区
# 输入t，修改某分区类型
# 更多命令看相关资料

# 分区操作步骤
# 1. 输入n，添加一个新的分区，n是new的首字母
# 2. 输入p，p表示主分区，e表示扩展分区（比较小就几十M）
# 3. 根据提示选择分区编号
# 4. 选择大小区间，起始值。可以用Enter取默认起始值
# 5. 选择大小区间，结束值。可以用[+8G]形式，这样避免计算
# 6. 输入w，保存退出


# 同步磁盘，分区完毕后执行下，否则下面创建文件系统可能找不到
partprobe

# 文件系统类型有ext2、ext3(日志)、ext4、vfat、msdos、jfs、reiserfs
mkfs -t ext4 /dev/sdb2

# 配置系统启动自动挂载，否则重启就没了
echo `blkid /dev/sdb2 | awk '{print $2}' | sed 's/\"//g'` /yh ext4 defaults 0 0 >> /etc/fstab
# 查看配置
cat /etc/fstab
# 最后挂载分区到一个空目录下
mkdir /yh
mount /dev/sdb2 /yh

            # [swapon] 交换

            # 查看交换空间数据
free -h
swapon -s
# 查看swap利用率，0表示优先使用物理内存，一般设置30
cat /proc/sys/vm/swappiness

vi /etc/sysctl.conf

# 新建，方式一，用分区创建swap，参考fdisk分区操作
fdisk /dev/vdb
# 1. 输入n，添加一个新的分区，n是new的首字母
# 2. 输入p，p表示主分区，e表示扩展分区（比较小就几十M）
# 3. 根据提示选择分区编号
# 4. 选择大小区间，起始值。可以用Enter取默认起始值
# 5. 选择大小区间，结束值。可以用[+8G]形式，这样避免计算
# 6.1 输入t，修改分区类型
# 6.2 输入L，可以查看所有文件类型，可以发现82编号是Linux swap / Solaris
# 7，输入82
# 8. 输入p，查看分区情况
# 9. 输入w，保存退出

# 新建，方式二，新建文件形式，用dd命令
# if =输入文件（或设备名称），of =输出文件（或设备名称），count=大小，bs=单位
dd if=/dev/zero of=/swapfile count=2048 bs=1M
# 验证文件是否生成
ls / | grep swapfile
# 赋予权限，改为只有所有者的读写权限
chmod 600 /swapfile

# 格式化Swap，不管上面那种方式新建，都要格式化。假设分区是/dev/vdb1
mkswap /dev/vdb1
# 激活Swap
swapon /dev/vdb1
# 查询命令可以看到了
free -h

            # [top] 动态进程信息

            第一行|top：系统时间、开机时长、登录用户数量、1、5、15分钟的负载情况
第一行|进程统计：运行、休眠、停止、zombie状态（僵尸）
第三行|cpu情况：us（用户空间占比）、sy（内核空间占比）、ni（改变优先级进程占比）、id（空闲占比）、wa（io等待占比）、hi（硬中断占比）、si（软中断占比）
第四行|内存情况：物理总量、空闲总量、使用总量、缓存总量
第五行|交换分区：交换总量、空闲总量、使用总量、可用总量
最后|进程监控：其中pr（优先级），ni（负值为高优先级），virt（内存），res（未交换出的内存）

            # 重定向输出输入

            • 0：标准输入

            • 1：标准输出，控制台，常常忽略如>/dev/null即1>/dev/null

            • 2：错误输出

            • >：输出重定向，先删除文件再创建输入到该文件，

            • >>：输出重定向，文件存在（不存在则创建），直接追加到该文件末尾

            • <：输入重定向。<<同理

            • /dev/null：空设备，直接丢弃，不占空间

            • 例子

              # 后台挂起运行名 nohup [...] & 这处将2重定向给1，再都输出到空设备
nohup [执行命名] >/dev/null 2>&1 &

            # 云服务器外网环境部署

            容易被黑客扫描攻击，常见现象为：安装定时任务、异常任务占满CPU 注意事项：

            • 安装的软件需设置或修改默认密码和端口
            • sshd设置
              • 修改sshd默认端口；
              • 禁用密码登录，采用秘钥登录；
              • 设置root账户登录失败次数；
              • 设置内网ip登录
            • 安全组入规则接口开放 解决方式
            • top查看进程信息，记录异常进程号
            • lastlastlog查看异常登录的用户和ip
            • crontab -l -u 用户查看异常用户的定时任务
            • crontab -e -u 用户编辑删除异常定时任务，记下定时任务路径
            • kill -9 异常进程号杀死1中发现的异常进程号
            • rm -rf 异常任务目录删除5中发现的异常任务目录
