        Linux Cron 相关总结

        寒铁 2020-07-13 linux

        # 服务

        • 守护进程，即后台一直运行
        systemctl status crond.service
        1

        # 命令

        • 系统自带的的定时命令crontab，支持语法检测

        • 相关目录、文件

          • 日志文件：/var/log/cron
          • 黑名单文件：/etc/cron.deny
          • 定时文件目录，以用户名命名：/var/spool/cron
          • 每月、每周、每日、每小时的脚本目录，默认存放系统脚本
            • /etc/cron.monthly
            • /etc/cron.weekly
            • /etc/cron.daily
            • /etc/cron.hourly

        • 基本命令

          crontab --help
# 查看当前用户，与用户挂钩
crontab -l
cat /var/spool/cron/root
crontab -l -u [用户] 
 
# 编辑，类似vim用法，保存在当前用户的定时任务脚本中
crontab -e

# 查看日志
tail -f /var/log/cron

        • 定时任务格式

          分 时 天 周 月
* * * * *
不支持秒级，可通过脚本自定义执行，循环执行sleep

秒级脚本参考
while true; do
  echo "hello world"
  sleep 1
done 

*， 就表示每分钟
00， 就表示分钟时刻为00，分0-59，时0-23，天1-31，周0-7（0和7表示周日，支持英文简写）
*/3， 就表示每3分钟
30,40 就表示30分钟和40分钟

eg：每天8点、18点运行，注意分级为0或00
0 8,18 * * *

        • 脚本输出

          • 不加输出，时间久了容器造成磁盘空间不足问题

          • 默认定时任务执行一次，会发送一次邮件给用户，路径/var/spool/mail

          # 将前面结果输出(1)重定向到指定文件，文件须提前创建
>/home/cron/test.log

# 将前面结果输出(1)重定向到垃圾箱
>/dev/null

# 标准错误输出（2）重定向到标准输出(1)
2>&1

        # 问题

        • 非root用户的cron任务，调用的sh脚本中涉及到sudo命令，需要配置用户的免密权限。

          # root用操作
# vi /etc/sudoes

# 修改 yh用户 支持sudo命令 
yh  ALL=(ALL)       ALL
# 修改 yh用户免密
yh  ALL=(ALL)       NOPASSWD: ALL
