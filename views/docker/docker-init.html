# 镜像加速

出现以下错误，都是网络连接问题

Error response from daemon ......... connection reset by peer

        "https://registry.docker-cn.com",

# 私有仓库

新版docker要求，私有仓库必须用https协议！

第三方软件，比如portianer，支持添加私有仓库，无需重启docker。

若自己搭建，比如nexus3的仓库，只有http端口，则需要在daemon.json中指定私有仓库地址！

{
  "insecure-registries": ["XX.91.199.54:8212"],
}
1
2
3

若设置权限的话，则推送和拉取需要先登录

docker login --username=xxx xx.91.199.54:8212
1

私有仓库的镜像，前缀都是私有仓库名，比如xx.91.199.54:8212/nginx:alpine

# Docker Desktop

  • window版，Docker Desktop 官方文档 (opens new window)

  • 打开Hype-V

    • 家庭版没有Hype-V，所以先安装这个虚拟器程序

    • cmd命令窗口执行systeminfo，查看处理器是否支持，一般都支持

    • 新建文本，复制脚本，修改后缀为cmd运行，自动联网安装，重启电脑

      pushd "%~dp0"
dir /b %SystemRoot%\servicing\Packages\*Hyper-V*.mum >hyper-v.txt
for /f %%i in ('findstr /i . hyper-v.txt 2^>nul') do dism /online /norestart /add-package:"%SystemRoot%\servicing\Packages\%%i"
del hyper-v.txt
Dism /online /enable-feature /featurename:Microsoft-Hyper-V-All /LimitAccess /ALL
      1
      2
      3
      4
      5

    • 修改注册表计算机\HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion 的Edition的数据值为Professional

      修改后就可安装。但注册表信息重启电脑就会恢复，后续有时会无法启动服务器，即提示daemon is not running ,需要重新修改注册表，再重启。家庭版Windows问题还挺多，建议升级系统

# DockerToolBox

是安装VM虚拟机，虚拟机里再安装docker。

注意虚拟机的的IP、端口、存储、共享目录

window的cmd窗口可能无法连接获取IP，用git的bash窗口

  • 官方参考 (opens new window)、博客参考 (opens new window)、疑问参考 (opens new window)、疑问参考 (opens new window)

  • 下载软件： git (opens new window)

  • （可选）修改虚拟机磁盘，默认路径（C:\Users\{myUsername}\.docker\machine\machines\)

    • 一般配置的虚拟机磁盘都是动态分配，后期docker镜像新增，占用系统盘空间
    • 配置用户变量MACHINE_STORAGE_PATH，必须重启电脑

  • （可选）移动虚拟机磁盘，适合某个虚拟机

    • 默认创建的磁盘C盘，关闭虚拟

    • 管理-虚拟介质管理-修改磁盘路径，选择磁盘，保存到新路径下disk.vmdk

    • 新增，选择新路径下的磁盘

  • docker-machine 命令参考

    # 创建docker的虚拟机

# 进入虚拟机系统，default默认创建的虚拟机名
docker-machine ssh default
# 可以用VB软件直接操作
docker-machine stop|start|restart default
