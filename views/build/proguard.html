# ProGuard

# 简介

ProGuard 是一个免费的 Java类文件的压缩，优化，混肴器。

通过制定的规则将java编译代码二次压缩优化或混淆，达到简化代码、混淆代码的效果，统称混淆。

对每个类引入一个Entry Point（入口点）概念，执行分四个阶段

  • 压缩（Shrink）阶段：从入口开始递归遍历，根据规则是否移除类、类成员
  • 优化（Optimize）阶段：没有入口的类，进行字节码级别的优化，比如设置为private、static或final，删除未使用的参数，方法标记内联。
  • 混淆（Obfuscate）阶段：对类、字段、方法进行重命名
  • 预检（Preveirfy）阶段：对处理后的代码进行预检，确保加载的class文件是可执行的

# 常用规则

  • keep：指定类和类成员（字段，方法）作为入口点被保留

    # 只保留类名
-keep public class *
# 保留类和类的内部代码
-keep public class * { *; }
# 保留类中某些数据，比如公开的元素
-keep public class * { 
      public protected *; 
}

# 使用方案

  • 从Github (opens new window)下，运行/bin/proguardgui.bat，直接对jar包进行混淆。参考 (opens new window)、参考 (opens new window)

  • maven插件，Github文档 (opens new window)

    <plugin>
    <groupId>com.github.wvengen</groupId>
    <artifactId>proguard-maven-plugin</artifactId>
    <version>2.3.1</version>
    <dependencies>
        <dependency>
            <groupId>net.sf.proguard</groupId>
            <artifactId>proguard-base</artifactId>
            <version>6.2.2</version>
        </dependency>
    </dependencies>
    <executions>
        <execution>
            <!--打包时执行混淆 -->
            <phase>package</phase>
            <goals>
                <goal>proguard</goal>
            </goals>
        </execution>
    </executions>
    <!--配置信息-->
    <configuration>
        <!--混淆时的内存配置-->
        <maxMemory>768m</maxMemory>
        <!--  是否将生成的PG文件安装部署 -->
        <attach>true</attach>
        <!--  是否混淆 -->
        <obfuscate>true</obfuscate>
        <!--  指定生成文件分类  -->
        <attachArtifactClassifier>pg</attachArtifactClassifier>
        <!--引入规则配置文件-->
        <!--<proguardInclude>${basedir}/proguard.conf</proguardInclude>-->
        <!--直接引入规则配置-->
        <!--<options>.....</options>-->
        <!--单独指定依赖，以下jar不在依赖关系中，比如JDK-->
        <libs>
            <lib>${java.home}/lib/rt.jar</lib>
            <lib>${java.home}/lib/jsse.jar</lib>
        </libs>
        <!--  要处理的jar -->
        <injar>${project.build.finalName}.jar</injar>
        <outjar>${project.build.finalName}-small.jar</outjar>
        <!--  输出目录 -->
        <outputDirectory>${project.build.directory}</outputDirectory>
    </configuration>
</plugin>

    其中引入规则配置文件progurard.conf如下

    # Proguard 配置项
# JDK版本
-target 1.8
# 默认开启，这里关闭shrink，即不删除没有使用的类/成员（删除注释、未被引用代码）
-dontshrink
# 默认是开启的，这里关闭，不作字节码级别的优化（变更代码实现逻辑）
-dontoptimize
-keepdirectories
# 不路过非公用类文件及成员
-dontskipnonpubliclibraryclasses
-dontskipnonpubliclibraryclassmembers
# 优化时允许访问并修改有修饰符的类和类成员
-allowaccessmodification
# 确定统一的混淆类的成员名称来增加混淆
-useuniqueclassmembernames
# 保留所有包名，Spring配置中有大量固定写法包名
-keeppackagenames
# 保留特殊类：对异常、注解信息在runtime予以保留，不然影响springboot启动
-keepattributes Exceptions,InnerClasses,Signature,Deprecated,SourceFile,LineNumberTable,LocalVariable*Table,*Annotation*,Synthetic,EnclosingMethod
# 保留所有set/get方法
-keepclassmembers public class * {void set*(***);*** get*();}
# 保留所有接口
-keep interface **.*{ *;}
# 保留类名，Bean注入问题
-keep class **.*
# 保留所有枚举
-keep enum **.*{ *;}
# 保留序列化类
#-keep public class * implements java.io.Serializable{   public protected private *; }


# 混淆类名之后，对使用Class.forName('className')之类的地方进行相应替代
#-adaptclassstrings
# 忽略打包时的警告信息
#-ignorewarnings

# 保留参数名
#-keepparameternames

# 保留main方法
#-keepclasseswithmembers public class * {
#    public static void main(java.lang.String[]);
#}
# -------------------------具体项目配置------------------------------
# 保留需要的类，否则影响项目运行
-keep class com.yh.*.*.controller.**
-keep class com.yh.*.*.mapper.**
# -------------------------具体项目配置------------------------------

# 输入|输出

maven配置中有2个标签，作用是混淆的输入输出目录

                    <!--需要作混淆的jar或classes目录-->
                    <injar>${project.build.finalName}.jar</injar>
                    <injar>classes</injar>
                    <!--混淆输出的jar-->
                    <outjar>${project.build.finalName}-pd.jar</outjar>
