# docker安装

version: '3'

services:
  nexus3:
    # 容器名
    container_name: nexus3
    # 重启策略: 容器退出时总是重启容器
    restart: always
    # 默认最新版本
    image: sonatype/nexus3
    ports:
      - 9777:8081
    volumes:
      - /home/soft/nexus3/data:/nexus-data

# 基本信息

  • 仓库里包存储形式：二进制文件

  • 默认存储路径: /home/soft/nexus3/data/blobs/default/content

  • 默认已创建Maven仓库和Nuget仓库(.NET)例子，可直接用

# Maven仓库

管理员角色登录可配置，默认已提供几个仓库，如图

  • maven-central：即中央仓库，proxy，代理仓库
  • maven-public：默认仓库组，group，实际使用的仓库地址，包括顺序如下
    • maven-central
    • maven-releases
    • maven-snapshots
  • maven-releases：正版仓库，hosted
  • maven-snapshots：快照仓库，hosted

# 镜像仓库

根据以上理解，新增阿里云镜像仓库， 参考阿里云仓库服务 (opens new window)

并且将仓库添加到maven-public组，顺序为第一个

# 实际应用

  1. 私有镜像仓库，添加maven-public组的地址

    • 本地Maven配置setting.xml

      <mirror>
    <id>my nexus</id>
    <mirrorOf>*</mirrorOf>
    <name>nexus</name>
    <url>http://localhost:8081/repository/maven-public/</url>

# Docker仓库

# 镜像仓库

  • 可选，新增个Blob Stroes，区分其他仓库的二进制存储路径

  • 创建三个Repositores

    • docker-release：hosted，私有仓库，用于存储自己的镜像

    • docker-xx：proxy，私有镜像仓库，可配置其他镜像仓库，同步到私有镜像仓库中，索引选择Docker-Hub保证获取的镜像是最新的

    • docker-public： group，仓库组，配置好以上仓库的顺序

    主要是hosted，其他可选，可直接用公开的镜像仓库如阿里云

  • 激活Docker Rleam

# 实际应用

  • Linux，修改`/etc/docker/daemon.json守护进程文件，添加私有镜像仓库和私有仓库地址

    {
  "registry-mirrors": ["https://registry.docker-cn.com"],
  "insecure-registries": ["192.168.0.82:9776"],
  "debug": true,
  "experimental": true
}
