      多账户SSH配置

      寒铁 2020-02-28 git

      # 简介

      user.name 和 user.email 信息作为git提交代理的用户信息

      git 的配置分为三级别：

      • System 即系统级别
      • Global 为配置的全局
      • Local 为仓库级别

      优先级是 Local > Global > System

      因此多账户下，建议清除全局配置，采用local配置，每个项目仓库配一次

      单账户直接用全局配置，不用后面的config文件

      # 查看修改配置

      #查看配置：全局、本地和系统
git config --global --list
git config --local --list
git config --system --list
#清除全局用户信息
git config --global --unset user.name
git config --global --unset user.email
#设置本地信息，在项目克隆或初始化后执行
git config --local user.name "yanghan"
git config --local user.email "yanghan"
#查看本地仓库配置
git config --local --list

      # 多账户配置

      秘钥同目录（.ssh目录）下，新建config文件，无后缀，配置如下

      # github.com
Host github.com
HostName github.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_rsa

# gitlab.company.com
Host 192.168.0.82
HostName 192.168.0.82
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_rsa_fw

      该文件分为多个用户配置，每个用户配置包含以下几个配置项：

      • Host：仓库网站的别名，随意取，建议直接和hostName一样
      • HostName：仓库网站的域名（PS：内网IP 地址也可以）
      • User：仓库网站上的用户名
      • IdentityFile：私钥的绝对路径

      注： Host 就是可以替代 HostName 来使用的别名，如git@{Host或HostName}

      实际应用，本地项目仓库配置，用户邮箱，会自动识别

      git config --local user.name "xx"
git config --local user.email "xx@xx.com"
