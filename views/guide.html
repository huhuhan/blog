    # 快速上手一个项目

    1. 熟悉项目所用框架技术，熟悉其规定的语法糖基本用法，建议通过官网文档学习；高级用法暂时忽略
    2. 了解基本语法糖用法后，便可以阅读代码，梳理业务逻辑，建议整理代码模块的引用，添加代码注释
    3. 基本业务逻辑整理后，可以进行具体调试，细节了解具体模块；
    4. 具体模块中若用到高级用法、疑点等再查阅资料，这类一般通过博客资料快速查阅，有官方文档优先官方

    # 传参加密意义

    1. 优先HTTPS传输协议，保证客户端到服务端的数据加密。
2. 传参加密
	可有效避免【社会学攻击】（即知道了源密码去试不同网站，假设密码相同）。
	保护了源密码，但可用直接用密文登录，除非后端解密要求时效性。
	类似代码混淆，深入前端js的代码逻辑，可得知加密步骤
3. 所以重点在时效性验证，比如时间戳、验证码，一起传参验证
4. 是否生态需求选择
	比如Github的登录密码没有加密，可用模拟登录。可用扩展第三方应用
	比如Gitee的登录密码前端加密，无法模拟登录，但有时效性

注意区分，密码存储的加密方式

    # 响应时间

    1. 响应时间是指系统对请求作出响应的时间。
2. 通常是指该系统所有功能的平均时间或者所有功能的最大响应时间。
3. 软件性能的高低实际上取决于用户对该响应时间的接受程度。

    # 吞吐量

    - 吞吐量Throughput是指系统在单位时间内处理请求的数量。
- 并发系统，通常需要用吞吐量作为性能指标。
- 单用户系统，响应时间（或者系统响应时间和应用延迟时间）可以很好地度量系统的性能。

    # 并发用户数

    - 并发用户数是指系统可以同时承载的正常使用系统功能的用户的数量。
- 一个更直观但也更笼统的性能指标。以在线用户作为性能指标更直观些，而以同时发请求用户数作为性能指标更准确些。

    # QPS每秒查询率

    - 每秒查询率QPS是对一个特定的查询服务器在规定时间内所处理流量多少的衡量标准
- 即每秒的响应请求数，也即是最大吞吐能力
- TPS，每秒事务数量， （单接口事务：发送请求、处理请求、响应请求），单接口情况下tps=qps

    # Java平台

    Java 分类：
  Java SE：标准版，之前叫J2SE
  Java EE：企业版，之前叫J2SE
  Java ME：微型版，之前叫J2SE
 
Java 应用包
  JDK是 Java SE SDK （Soft Development Kit) 软件开发工具包的扩展子包，针对java开发应用。
  JDK(Java Development Kit)，包含JRE，同时用于开发编译、调试等功能。
  JRE(Java Runtime Enviroment)，即java程序运行需要的环境包，其中包括Java虚拟机、Java平台核心类和支持文件。
  
Java 版本：
  最开始是JDK 1.1、1.2
  后改为J2SE 到 1.5 版本
  又改为J2SE 5、6
  J2SE、J2EE、J2ME 统一改为 Java SE、Java EE、Java ME
  所以JDK 1.8 就是 Java SE 8 简称 Java 8

Java 标准规范
  Java是个大平台，提供开发用的JDK，包括基础的类、标准接口等等，具体实现由第三方去扩展。
  比如JDK中：JMS消息服务、JDBC数据库连接、JTA事务API、XML可扩展标记语言、Java Servlet、JSP、JNDI、EJB
  这些都是Java的标准规范，每次JDK版本发布有一套对应的规范，大致规范变动不大，所以简化版本称呼。
  以上这个统称为Java SE规范，对应还有Java EE规范等等

  Java各种标准的制定是通过Java Community Process 组织即JCP发布的。
  JCP的成员可以根据需要提出Java Specification Request，即JSR。
  类似Github上的项目管理，请求分支合并。
  比如Java EE标准规范就是一组被通过的JSR的合集

    # LRU

    - 算法：最不经常使用，Least Recently Used
- 一般适用于缓存的算法
- 存储的内存有限，所以存储的是相对价值高的数据，根据算法来更新更新内存中的数据。
- 用最近一次使用的时间来衡量一块内存的价值，最近刚使用过，接着被用的概率相对高点
- 此算法就是更新内存中最不经常使用的数据，相对来说不大可能被使用到的数据
- 缓存功能：1、查询、2、更新，综合时间空间复杂度，数据结构应该是HasMap+LinkedList
- HasMap，key：查询key，value：链表节点；
- 链表，假设头部为旧数据，尾部为新数据
- 情况分析：
  - 查询
    1. 查不到key，返回空
    2. 查到key，value在链表中移动到尾部（即，先删除节点，再插入节点）
  - 更新
    1. 已存在，更新HashMap值，链表移动，参考查询2
    2. 不存在
      - 缓存的内存还没到上限，直接插入链表尾部
      - 缓存的内存已到上限，删除头1，再插入链表尾部，同时删除HashMap的key
- 以上分析，用相关语言实现，注意链表要自己实现，HashMap就是最终的LRU

    # clickHouse

    -  Yandex（俄罗斯最大的搜索引擎）的开源**列式**数据库。
- 对比传统**行式**存储数据库（OLTP），处理速度快100-1000 ，适用于数据分析（OLAP）
- OLAP场景特点：
  - 读大于写
  - 大宽表，读大量行但是少量列，结果集较小
  - 数据批量写入，且数据不更新或少更新
  - 无需事务，数据一致性要求低
  - 灵活多变，不适合预先建模
- **列式**数据库优点
  - 只需要读取要计算的列数据，而非行式的整行数据读取，降低 IO cost。
  - 同列同类型，有十倍压缩提升，进一步降低 IO。
  - 根据不同存储场景，做个性化搜索算法。

    # JMS

    • Java技术规范，协议，类似JDBC，为消息通信定义了规范接口

    • 规范的概念有

      Provider(MessageProvider): 生产者
Consumer(MessageConsumer): 消费者
PTP: Point to Point，点对点的消息模型，一个消息只能被一个消费者消费
Pub/Sub: publish/Subscribe，发布/订阅的消息模型，一个消费被所有订阅的消费者消费
ConnectionFactory: 连接工厂，创建连接
Connection: 客户端到Provider的连接
Destination: 消息的目的地
    Queue: 队列目标
    Topic: 主题目标
Session: 会话，一个发送或者接受的线程

    # CAP

    1、针对分布式环境的理论
2、保证可用性，一个a节点数据会有N个副本，数据同步到另个b节点一般用复制形式。
3、因某原因无法同步，ab两节点数据就形成隔离，即分区容错性P(partition)
4、解决这个现象，两种选择
5、可用性 A(Availability)，即b点还可用，但数据不保证最新，没有即使同步a的数据
6、一致性C(Consistency)，即b点等待响应，直到数据同步完成
例子：
1、服务注册中心，需要记录各个微服务节点信息，会自动同步更新节点信息
2、Eureka，完全AP，同个服务可以注册多个eureka中，即注册中心集群
3、Zookeeper，完全CP
4、Nacos，支持AP或CP，集成配置中心
5、数据库读写分离

    # zookeeper

    主要服务于分布式系统
​	应用：统一配置管理、统一命名服务、分布式锁、集群管理
​	数据结构：
​		树形态，节点概念(Znode)
​		节点，一个连接创建一个节点，分：
​			短暂（断开连接，节点删除）
​			持久（断开连接，节点不删除）
​	监听器：
​		监听节点数据变化
​		监听子节点的增减变化
​	分布式锁例子：
​		一个应用访问创建一个短暂性节点，带编号，并监听子节点的增减变化；
​		判断规则以最大或最小的编号，为当前获取锁的依据；
​		释放锁，当前节点删除，最大（小）编号发生变化，子节点发生增减变化；
​		其他应用访问创建的节点监听到变化，重新判断，最大（小）的获取锁；
​	模式：
​		独立（单机)模式
​		仲裁（集群）模式：法定人数概念，建议集群数为奇数，如5台服务器，法定为3台，数据同步了3台就可响应客户端，即使崩溃2台，还能拿保证有一台可用；
​	技术问题：
​		1、启动失败？
​			参考zkEnv配置脚本，确定是否为zoo.cfg配置文件
​			java环境变量路径最后是否有多余的斜杆

    # LDAP：

    一种通讯协议，支持TCP/IP。协议就是标准，是抽象的。
LDAP是一种开放Internet标准，跨平台的Interent协议。
通常说法，就是指运行这个数据库的服务器，即存储数据的数据库。
区别于关系型数据库，是树型结构的存储数据，树形展示数据。
server/client模型：服务器存储数据，客户端操作信息树。

    # Redux

    js的状态容器，做状态管理，类似观察者模式或订阅发布模式
	核心概念：
		store：一个Redux一个store，可看做状态容器对象，通过该对象操作状态数据
		state：存储状态数据，类型可为基本类型、数组、对象
		action：描述发生了什么，有利于日志、调式等等
		reducer：一个纯函数，接收旧的state和action，处理后返回新的state对象
	注意：state是只读，所以只能通过action触发reducer更新数据并返回新的state对象
	适用：任何框架
	数据流：单向
	对比：
		Vuex：简化版Redux，简化了部分定义，Vue框架专门的状态容器(双向数据流）
	store的应用:
		提供 getState() 方法获取 state；
		提供 dispatch(action) 方法更新 state；
		通过 subscribe(listener) 注册监听器;
		通过 subscribe(listener) 返回的函数注销监听器。	
	目录结构
		1、
		--redux
		--------store.js
		--------modules
		----------m1.js		//包含state、reducer、action、mapStateToProp（react-redux的connect的过滤方法）
		----------m2.js
		2、
		--store
		--------store.js	//标准store，通过此处的构造函数返回store
		--actions
		----------a1.js		//按组件命名action
		----------a2.js
		--reducer
		----------r1.js		//按组件命名reducer
		----------r2.js
		其他直接在组件jsx里定义
	衍生中间件
		React-Redux：配合Redux使用
		reselect：缓存state，redux的性能优化；正常state发生变化，会触发所有的reducer；reselect为某组件的state做了个缓存，根据参数函数判断是否触发变化重新渲染；类似vue的computed
				
React—Redux：
	1、绑定store对象
	<Provider store={store}>
		根组件
	</Provider>
	2、组件订阅store
	connect()
		参数1：
			对象：从store中筛选返回当前组件所需的state，作为props属性；	类似Vuex的mapGetters
			函数：在筛选的state属性做个函数处理（比如过滤）再返回
				参数1：state
				参数2: ownProps
		参数2：从store中筛选返回当前组件所需的action，作为事件操作, 触发方式props.dispatch(action)

    # tomcat

    	1、下载解压版
	2、乱码问题，系统、服务编码不统一导致，建议都改为UTF-8
		1、添加（修改）启动java参数，-Dfile.encoding=UTF-8 -Dsun.jnu.encoding=UTF-8
		2、若启动日志乱码，\conf\logging.properties中java.util.logging.ConsoleHandler.encoding修改UTF-8为GBK(仅限window本地使用）
	3、多个tomcat
		修改端口，server.xml配置对应的8005、8080、8009
		若没有配置环境变量，直接可用。原理 startup脚本调用catalina脚本，找tomcat环境变量，没有就默认取当前目录。
	4、启动缓存报错
		修改conf下的context.xml，填下如下代码
		<!-- 默认开启缓存，提示缓存不够时，可以设置缓存大小，或关闭缓存 -->
		<Resources cachingAllowed="false"></Resources>
		<!-- <Resources cachingAllowed="true" cacheMaxSize="100000" /> -->

    # webpack

    配置
	entry，配置要打包的js（value）以及打包后的命名(key)
	output，配置打包后js存在路径，使用entry的key就取[name].js
	plugins：引用插件实现额外的功能，常用如下
		ExtractTextPlugin：单独分离css单独打包，通过link引入，否则css样式会打包在js以style标签直接加载head中
		HtmlWebpackPlugin：生成html入口文件；为html文件外部资源文件，如script、link动态添加每次compile后的hash，防止引用缓存的外部文件问题
		DefinePlugin：添加配置文件，构建期间自动检测环境变化
		ParallelUglifyPlugin：压缩插件多线程并行压缩，wabpack默认是单线程压缩，
    module: webpack默认只能处理js文件，其他文件需要通过loader加载，以下是模块加载规则定义
		noParse: [/html2canvas/], //忽略js，比较JQuery之类的没有模块且比较大的文件
		rules: [{
                //正则匹配某类型文件,test、
                test: /\.jsx?$/,
                //默认搜索编译的所有的文件，所以排除node_modules目录下文件
                exclude: /node_modules/,
                //指定用什么loader加载，可配置多个,支持对象，或直接loader的字符串
                use: [{
                    loader: "babel-loader",
                    options: {
                        cacheDirectory: true,
                    },
                }],
                // include:
            }]
	
写法	
	require同步加载
	require.ensure 异步加载js
	require('./')	当前目录
	require('../') 	父目录
	require('path')	node的核心模块或第三方模块;引用的是文件夹，默认找目录下的index.js
