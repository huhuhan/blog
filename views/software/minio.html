    # MinIO-Client

    • 可以操作Minio、管理多个服务节点。
    • 这里主要记录多用户权限控制的使用方式
    • 参考官方文档 (opens new window)

    官方最新推出的MinIO-Console (opens new window)就是可视化的控制平台

    # 安装

    # 下载镜像
docker pull minio/mc
# 后台运行
docker run -it --entrypoint=/bin/sh --name minio-client -d  minio/mc
# 进入容器
docker exec -it minio-client /bin/bash
# 验证
mc ls

    # 查看当前客户端的配置信息

# 新增配置文件，这里别名就叫yhminio
mc config host add yhminio http://xx.xx.xx.xx:9000 '账户' '密码'
# 测试
mc alias ls
# 直接查询，列出yhminio服务的所有bucket，其他操作命令参考文档
mc ls yhminio

    • Version 值是固定2012-10-17，不要修改
    • Effect 就两种：拒绝Deny、允许Allow
    • Action 这里列出的几种常用的，更多参考文档S3 Policy (opens new window)

    # 新增用户、组

    针对某用户权限的话，先创建个用户。

    # 新增用户demouser，密码demo1234 到 yhminio。注意密码至少8位
mc admin user add yhminio demouser demo1234
# 查看，看不到初始化管理员账户的
mc admin user list yhminio


# 查看信息
mc admin user info yhminio demouser
# 用户启用
mc admin user enable yhminio demouser
# 用户禁用
mc admin user disable yhminio demouser
# 删除用户
mc admin user remove yhminio demouser

    # 添加权限策略

    操作基于mc admin的policy、user命令，更多参考文档 (opens new window)

    首先根据需求创建一个自定义权限策略文件，假设叫demoPolicy.json

    {
	"Version": "2012-10-17",
	"Statement": [
		{
			"Effect": "Allow",
			"Action": [
				"s3:ListAllMyBuckets",
				"s3:ListBucket",
				"s3:GetBucketLocation",
				"s3:GetObject",
				"s3:PutObject",
				"s3:DeleteObject"
			],
			"Resource": [
				"arn:aws:s3:::demo/*"
			]
		}
	]
}

    默认都有4种策略值：diagnostics、readonly、readwrite、writeonly

    # 查看 minio 的当前策略，策略可针对每一级目录
mc admin policy list yhminio
mc admin policy list yhminio/demo
mc admin policy list yhminio/demo/test

# 将demoPolicy.json 先复制到容器中某个目录，这里mc目录记得先创建
docker container cp /yh/soft/minio/demoPolicy.json minio-client:/mc
# 容器内，策略文件路径
ls -l /mc

# 映射策略文件，直接赋值给minio，这里别名叫demoonly。映射后源文件删除不影响
mc admin policy add yhminio demoonly /mc/demoPolicy.json 
# 再次查看就有5种了
mc admin policy list yhminio
# 查看策略信息
mc admin policy info yhminio demoonly

# 删除策略
mc admin policy remove yhminio demoonly
