# 还原数据

  • 新增act_run_task记录

    • ID_：【引擎流程任务】自建（必须表中唯一，如421381212653228244）
    • RED_：1
    • EXECUTION_ID_：【引擎流程实例】
    • PROC_INST_ID：【引擎流程实例】
    • PROC_DEF_ID_：【引擎流程定义】
    • NAME_：【节点名称】
    • TASK_DEF_KEY_：【节点编号】
    • PRIORITY_：50
    • SUSPENSION_STATE_：1
    • CREATE_TIME_：【自建】

  • 新增bpm_task记录

    • id_：【业务流程任务】=【引擎流程任务】=（等于上面自建的编号)
    • name_：【节点名称】
    • subject_：【主题】
    • inst_id_：【业务流程实例】
    • task_id_：【业务流程任务】
    • act_inst_id_：【引擎流程实例】
    • act_execution_id：【引擎流程实例】
    • node_id：【节点编号】
    • def_id：【业务流程定义】
    • assignee_id_：0
    • assignee_names_：【节点人员】
    • status_：NORMAL
    • priority_：50
    • task_type_：0
    • parent_id_：0
    • create_time_：【自建】
    • create_by_：【自建】

  • 新增bpm_task_opinion记录

    • id_：【自建】（必须表唯一）
    • inst_id_：【业务流程实例】
    • sup_inst_id_：0
    • task_key_：【节点编号】
    • task_name_：【节点名称】
    • assign_info_：【节点人员当前候选】，注意格式，【role-名称-id】，多个逗号区分，参考下面bpm_task_identitylink表记录，可以先找个正常流程的历史记录复制下
    • status_：awaiting_check
    • dur_ms_：0
    • create_by_：【自建】

  • 更新act_run_execution记录

    • REV_：对应节点序号（当前例子，缺失节点是第8个，改为8）
    • ACT_ID_：当前【节点编号】（当前例子，改为UserTask8）
    • IS_ACTIVE_：改为1
    • SUSPENSION_STATE_： 改为1
    • CACHED_ENT_STATE_： 改为2

  • 新增bpm_task_stack记录，2条

    • id_：【自建】
    • inst_id_：【业务流程实例】
    • node_id：【节点编号】，注意非任务节点，如网关、节点跳转
    • node_name：【节点名称】，注意非任务节点，如网关、节点跳转、开始节点
    • parent_id_：上个记录的id，参考字段node_name
    • task_id：【业务流程任务】=【引擎流程任务】=（等于上面自建的编号)，注意非任务节点，统一填开始节点的task_id_
    • start_time：自建，建议按时间顺序来，方便查询对比

    这里建议查询其他正常流程实例，复制记录，再修改id_、inst_id_、task_id_

  • 新增bpm_task_identitylink，N条，当前节点候选记录，根据配置的节点人员而定

    • id_：【自建】
    • task_id_：【业务流程任务】=【引擎流程任务】=（等于上面自建的编号)
    • inst_id_：【业务流程实例】
    • type_：role（一般节点人员配置的都是角色，用户的话填user）
    • identity_name_：对应org_role表的id（用户对应org_user表）
    • identity_：对应org_role表的name（用户对应org_user表）
    • permission_code_：格式【identity-permission_code】
