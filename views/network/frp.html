      内网穿透-FRP

      寒铁 2020-08-07 network

      # 简介

      • 内网穿透，也即 NAT 穿透，进行 NAT 穿透是为了使具有某一个特定源 IP 地址和源端口号的数据包不被 NAT 设备屏蔽而正确路由到内网主机。即客户端A通过公网服务器代理访问到内网客户端B。
      • FRP (opens new window)，是一个可用于内网穿透的高性能的反向代理应用，支持 tcp, udp 协议，为 http 和 https 应用协议提供了额外的能力，且尝试性支持了点对点穿透。

      # 多个HTTP连接

      # 方法一

      公网服务器运行多个frps，一个frps支持一个HTTP连接。（测试可行，浪费多余端口不推荐）

      # 方法二

      还是由vhost_http_port端口(8298)实现穿透，但多个http连接，通过域名解析。

      # 多个HTTP端口配置，需要配合域名解析
[web-01]
type = http
local_port = 9777
# 子域名
custom_domains = test1.yhtt.com

[web-02]
type = http
local_port = 9776
custom_domains = test2.yhtt.domain

      # winsw服务

      window环境可以配置成系统服务，参考winsw (opens new window)，客户端配置参考如下

        <service>
      <id>frp</id>
      <name>frp-service</name>
      <description>frp远程连接桌面</description>
      <executable>%BASE%/frpc.exe</executable> 
  	<arguments>-c frpc.ini</arguments>
      <onfailure action="restart" delay="60 sec"/>
      <onfailure action="restart" delay="120 sec"/>
      <logmode>reset</logmode>
      <logpath>%BASE%/logs</logpath>
      <log mode="roll-by-time">
          <pattern>yyyyMMdd</pattern>
      </log>
  </service>

      # 支持远程桌面

      上面介绍了远程桌面的内网穿透例子，但Window家庭版默认是没有远程桌面功能。

      需要安装应用软件：

      • 下载 (opens new window)安装RDP软件，目录随意，其中

        RDPConf.exe		用于检查，是否支持远程连接功能，全绿色即可
RDPCheck.exe	用于测试能否远程连接，即电脑A连电脑A
        1
        2

      • 运行后会在C:\Program Files\RDP Wrapper目录生成RDP的配置文件，重点！

        • 根据windows版本不断更新，需要替换配置文件。当前版本RDPConf.exe中会显示。
        • 根据版本从库中 (opens new window)搜索对应配置（参考博客 (opens new window)）或加群 (opens new window)
        ## 先停止RDP服务
net stop termservice
## 替换配置文件 (C:\Program Files\RDP Wrapper)
## 再启动RDP服务
net start termservice
