        # 农村宅基地信息管理平台

        根据国家宅基地建房标准相关政策，为了加快农村宅基地建房工作，提高工作效率和服务水平，以数字化形式进行信息管理，支持各地区特色定制功能的一套完整宅基地信息管理平台。

        # 联合审批

        描述：负责平台中所有的审批业务

业务菜单
    业务预审（直接以业务流程命名菜单，通过菜单权限按需分配）
        建房申请：地区需求所致，先填表单数据，审核通过后再发起流程
        使用权流转
        退出抵押
    个人审批
        发起申请：当前用户可发起的流程
        我的待办：当前用户的待办任务
        我的已办：当前用户办理过的任务历史
        我的发起：当前用户发起的流程实例

业务预审的权限是正常控制，简单状态判断。
个人审批的权限是基于工作流引擎实现，在一体化控制中心平台提前配置好业务流程，分配给用户

其中流程的业务表单页面，大致内容如下：
	流程通用外框：显示通用按钮，比如流程图、任务历史、同意/驳回等等
	流程内置表单：即实际的业务表单，根据流程配置动态路由加载
	流程节点附件：右滑半窗口，管理当前节点的附件
	流程节点报表：右滑全窗口，管理当前节点的报表，报表页面是宅基地报表系统构建的独立页面

        # 执法监管

        描述：负责宅基地的巡查、执法、更改的流程

一般是app端应用，集合上面的业务流程比如建房申请
因为需求关系，没有应用工作流引擎，对业务数据的状态更新和信息补充实现流程

业务菜单：
	巡查上报
	执法派发
	执法取证
    执法审核
    整改措施
    整改验收
    归档统计
业务流程：
	大致流程：巡查举报=>执法派发=>执法取证=>执法审核=>整改措施=>整改验收=>归档
	1. 录入数据（标题、违法情况、违法地址、经纬度、巡查人员、巡查时间）
	2. 将违法数据，派发给执法人员，
	3. 执法人员，将取证数据录入
	4. 审核，执法数据和取证数据，通过
	5. 录入对应整改措施数据
	6. 对整改结果，验收归档
	7. 归档查询

        # 档案管理

        描述：汇总业务中的文件材料，作为档案管理

业务菜单:
	首页检索
	档案分类（直接以业务需求命名菜单，通过菜单权限按需分配）
        建房申请档案
        农户档案
		使用权流转档案
		有偿使用档案
		自愿退出档案
	档案销毁
	档案借阅

功能介绍
    各业务页面有对应的生成档案按钮；
    档案分类这的菜单是统一入口，都有生成档案按钮；
    生成档案操作，就是将该业务中的所有文件汇总出来；
	每份档案的文件，可自定义分类、再补充添加、删除（不影响原业务的文件）、打包下载；
	档案销毁后，可恢复（即逻辑删除）
	档案可借阅，需申请、审批后才有权查看，到期自动归还
	档案的所有操作都有日志记录（添加文件、删除文件、新加分类、删除分类、申请借阅、通过借阅申请、驳回借阅申请、销毁、恢复）


数据库表设计，都是通过fileId关联，file表统一维护文件信息，哈希值确定唯一文件

        # 项目功能设计

        大部分功能都以开放式设计来新增功能，最终由技术支持人员自由配置来满足多个地区版本不同的需求

        # 签名盖章

        • 盖章配置：签章图片、签名图片，关联地区、个人角色
        • 自动盖章：调取签章配置信息，一键盖章
        • 电子签名：
          • 可调取签名配置信息，一键签名
          • 可由前端插件（canvas绘画实现，支持硬件手写板），在线签名，生成图片完成签名
        • 电子签章：
          • 完成自动盖章后，后台生成PDF文件
          • 某流程节点触发定时任务，后台调用第三方接口，合成有法律效应的PDF签章文件，支持在线验签
          • 点聚电子签章系统（北京点聚信息技术有限公司），通过CAS认证的数字签章

        # 证书生成

        • 证书规则先通过证书配置菜单也配置，类似上述签章配置。
        • 这里主要定义生成规则，比如批准书号，编码位数+乡号+发证机构
        • 业务流程中用到，只需一键生成
        • 机构管理，配置机构信息

        # 路由菜单

        • 平台菜单，由前端开发后配置可访问的页面路由
        • 管理员授权角色可访问的路由菜单，动态渲染

        # 表单配置

        低代码设计，自定义业务表单，数据存储后端数据库，调取后前端动态渲染。减少开发工作，适用大部分业务。

        • 业务表单：网页端、手机端的简单表单设计
        • 业务对象：表单实体对象设计，后台数据库生成表，实现增删改查
        • 自定义对话框：窗口数据，主要是查询、选择功能

        # 流程配置

        基于Activiti工作流引擎实现，完全独立，与业务项目解耦，可由非开发人员自主配置

        • 流程配置主要步骤：
          • 流程图配置
          • 流程节点表单配置：
            • 选择系统表单：基于上面配置好表单数据
            • 选择路由表单：特殊定制，只需配置VUE路由菜单相对地址，表单页面数据在前端项目中
          • 流程节点候选人配置：支持个人、角色、组织
          • 流程节点操作配置：比如同意、驳回、转办、抄送等按钮功能，部分基于自定义对话框
          • 插件配置：比如消息通知、节点跳转规则等等
          • 流程授权：授权后，个人或某业务模块即可看到数据，即发起、待办、已办、历史等类似菜单

        # 附件上传

        流程插件，依赖于流程定义，需要先配置好业务流程图

        • 根据流程节点信息，可设置各节点需要上传的附件数据

        • 支持多级分类、数量限制、是否必传

        • 配置信息与业务流程定义关联，附件数据与业务流程实例关联

        • 使用时，各节点调取附件上传配置信息，需要按要求上传，才通过当前节点验证走到下个节点

        # 表单打印

        流程插件，依赖于流程定义，需要先配置好业务流程图

        • 基于ureport依赖包搭建的独立服务，根据业务需求定制表单
        • 打印配置：自由配置业务流程节点所需显示的报表
        • 业务呈现：调取报表配置，单独加载报表页面

        # 地图配置

        • 图层设置：影像注记、矢量底图等底图图层信息，控制是否默认加载
        • 工具设置：工具是否显示，比如放大、缩小、定位等等

        # 系统配置

          • 通用移动端app，初始化加载读取服务配置地址
        • 数据分类设置：根据递归树分类定义，业务中需要分类只需添加关联字段，按分类过滤
        • 数据字典设置：
        • 数据库源设置：多租户版本功能，根据租户信息，业务数据动态加载业务库
        • 任务调度配置：嵌入xxl-job-admin管理平台页面
        • 字体图标设置：页面风格简单自定义功能

        # 后端技术栈

        • 微服务分布式版本：Spring Boot 2.X + Spring Cloud + Spring Cloud Netflix系列组件
        • 服务治理注册发现：Spring Cloud Netflix Eureka
        • 统一网关路由：Spring Cloud Gateway
        • 声明式服务调用：Spring Cloud OpenFeign
        • 服务降级熔断：Spring Cloud Netflix Hystrix
        • 服务负责均衡：Spring Cloud Netflix Ribbon
        • 服务安全认证：Spring Security + Oauth2 + JWT
        • 数据库访问层：Mybatis、Mybatis-Plus
        • 服务应用监控：Spring Cloud Admin
        • 统一日志存储：ELK
        • 消息中间件：MQ（Active、Rabbit）
        • 分布式任务调度：XXL-Job
        • 搜索引擎：Elasticsearch
        • 数据库：PostgreSQL、MySQL
        • 缓存：Redis
