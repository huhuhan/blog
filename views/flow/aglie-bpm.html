AgileBpm 工作流总结

寒铁 2020-02-28 workflow

# AgileBPM

# 源项目

  • 开源项目 (opens new window)，于2019年停止维护，区分商业版对比参考 (opens new window)
  • 流程引擎基于Activiti-5.22版本开发
  • Maven模块化结构，其中流程模块，相关代码未提供，jar是代码混淆过的
  • 区别原生流程引擎，是通过流程引擎的事件进行操作，将操作记录在扩展的自定义表中

# 二次开发

  • 开源版，将核心模块（wf）代码反编译，并梳理逻辑重构，解决代码混淆问题

  • 项目重构为SpringBoot版本

  • 功能扩展和修复，实现部分商业版功能

    • 流程表单：支持Vue表单、插件窗口Vue组件化
    • 流程插件：催办、自由跳转、自由驳回、撤销、终止、转办、抄送
    • 流程模型：导入导出
    • 异常修复：流程实例丢件（开源版BUG），支持找回再自由配置（通过自由跳转实现）

  • 支持代码混淆，同样通过proguard实现构建混淆，源码暂不开放

  • 重构，分离式VUE前端项目 (opens new window)

  • 重构，扩展功能的后端项目 (opens new window)

# 项目分析

# Mybatis调用

Agile是采用原生Mybatis开发的，即源码的base-db模块。相对常用的Myabtis-plus等第三方组件，可能没有一些通用方法，需要自己实现。

定义了一套通用的增删改差的基类：

  • Controller层：BaseController和ControllerTools
  • Service接口：Manager

# 流程设计器

Agile将Activiti自带的流程设计器，通过AngularJS扩展一些交互窗口了，将流程配置交互更方面了，如图

这也是我看中的原因，原生自带的交互不好用，比如都是输入框，而不是下拉选择框。对AngularJS熟悉度不高，所以流程设计器前端这块仅针对接口的调用作了修改。

    【示例】任务抽象类，定义任务的执行流程 
    @Override
public void notify(ActivitiEvent event) {
    ActivitiEntityEvent entityEvent = (ActivitiEntityEvent) event;
    TaskEntity taskEntity = (TaskEntity) entityEvent.getEntity();

    // TaskEntity对象转为DefaultTaskActionCmd
    T model = getActionModel(taskEntity);

    // 执行前置插件之前
    beforePluginExecute(model);
    // 触发前置插件
    if (getBeforeTriggerEventType() != null) {
        this.taskCommand.execute(getBeforeTriggerEventType(), model);
    }

    // 触发具体事件操作
    triggerExecute(model);

    // 触发后置插件
    if (getAfterTriggerEventType() != null) {
        this.taskCommand.execute(getAfterTriggerEventType(), model);
    }
    // 执行后置插件之后
    afterPluginExecute(model);
}

# ApplicationListener

AbstractMessageQueue implements ApplicationListener<ContextRefreshedEvent>

抽象公告消息队列，实现ApplicationListener<ContextRefreshedEvent>接口
	onApplicationEvent():注册消息处理器接口，实现类的Map集合，存储内存中
		并提供初始化接口实例的后置方法
		注册实现类如下：
			1、同步队列，即生产立马消费，生产者=消费者
			2、Redis队列，生产者，以数组存储，消费者开启线程消费该数组对象。
					线程通过初始化接口实例的后置方式开启
			3、通用队列，基于MQ实现
					生产者通过JmsTemplate发送至MQ
					消费者消费，消费初始化配置MQ连接
