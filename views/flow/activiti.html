        Activiti 工作流总结

        寒铁 2020-02-28 workflow

        # 概念

        • 流程
          • 就是通过计算机对业务流程进行自动化管理，实现多个参与者按照预定义的流程去自动执行业务流程
        • 流程模型
          • 使用流程设计器定义好流程图保存后为流程模型，生成BPMN文件.
          • 部署发布后，生成流程部署和流程定义，两者一一对应。
          • 可重复部署，但生成的流程定义Key相同，版本号不同。推荐只部署一次。
        • 流程部署
          • 仅仅做为流程已部署可用的记录。
        • 流程定义：
          • 可根据定义Key发起流程实例后，关联最新版本的流程定义。
          • 可根据主键id发起流程实例，不推荐。
          • 可控制状态激活或挂起，影响所有发起的流程实例。
          • 可逆向转为流程模型。
        • 流程实例
        • 根据流程定义的key发起后，生产一个运行实例和历史实例。
        • 运行实例间不影响。走完流程，运行实例自动删除。
        • 运行实例可控制激活或挂起。
        • 历史实例：一直保留，除非手动删除。
        • 节点任务
          • 流程图中的节点任务userTask
          • 当前和历史的节点任务
          • 当前和历史的节点任务参与者（用户或组）
        • 流程变量
          • 流程图中的事件、任务或其他额外变量信息
        • 流程事件
        • 网关
          • 流程图设计器中需要使用的组件
          • 排他网关，分支选一，进入下个节点任务
          • 并行网关，分支并发进行，所有分支执行结束后，进入下个节点任务
          • 包含网关，排他和并行结合，部分分支符合条件才可执行，所有分支执行结束，进入下个节点任务
        • 定时器

        # 接口调用流程

        1. 创建模型
        即创建bpmn文件，简称【流程图】
获取modelId, 访问编辑器【/modeler.html?modelId=】，编辑流程图并保存
【流程图】以下参数存储在Variables对象中
	必填属性
		1. 总信息：流程ID（Process Identitifer）、名称（Name）、说明（Documentation）
		2. 开始事件：设置发起人（），本项目参数名固定为【applyUserId】，格式${applyUserId}
		3. 用户任务：设置代理人（Assignments）或候选人或组，必须是已同步到工作流数据库中的用户
		4. 判断事件：设置判断参数（），本项目参数名固定为【passFlag】, 格式${passFlag}，否为${!passFlag}
		5. 结束
	可选属性
保存后填写模型信息，获得Model对象
关键属性
	流程ID（Process Identitifer），部署后变为【流程key】ProcessDefinitionKey

        # 设计思路

        • 监听器接口设计

          1. 全局事件监听器接口：根据监听acitivit引擎的操作事件来触发自定义事件
          2. 抽象监听器接口
            • 流程实例抽象类
              • 实例创建
              • 实例结束
            • 流程任务抽象类
              • 任务创建
              • 任务结束
            • 流程发起
            • 流程完成
            • 流程取消
            • 连线创建
            • 定时执行
          【示例】任务抽象类，定义任务的执行流程 
          @Override
public void notify(ActivitiEvent event) {
    ActivitiEntityEvent entityEvent = (ActivitiEntityEvent) event;
    TaskEntity taskEntity = (TaskEntity) entityEvent.getEntity();

    // TaskEntity对象转为DefaultTaskActionCmd
    T model = getActionModel(taskEntity);

    // 执行前置插件之前
    beforePluginExecute(model);
    // 触发前置插件
    if (getBeforeTriggerEventType() != null) {
        this.taskCommand.execute(getBeforeTriggerEventType(), model);
    }

    // 触发具体事件操作
    triggerExecute(model);

    // 触发后置插件
    if (getAfterTriggerEventType() != null) {
        this.taskCommand.execute(getAfterTriggerEventType(), model);
    }
    // 执行后置插件之后
    afterPluginExecute(model);
}
