    # 启动失败

    windows环境

    • 原因：

      • 由于执行命令超时，导致事务锁住。

    • 状况：

      • 会在目录/data/pg_xlog/下会生成事务文件，删除文件。（删不掉提示权限问题，只能重启系统才可删）
      • 会在目录/data下生成pid后缀文件，删除文件

    • 命令查询

      • 重置事务日志

        pg_resetxlog.exe -f ..\data
        1

      • 查看服务状态

        pg_ctl.exe status -D ..\data

# 获取到pid，杀掉进程
taskkill -pid 3242

    # 查看表字段

    -- 查询自定义表my_user的字段数据
SELECT
	A.attname,
	T.typname 
FROM
	pg_class AS C,
	pg_attribute AS A,
	pg_type AS T 
WHERE
	C.relname = 'zgq_user' -- 主外键关联
	AND C.oid = A.attrelid 
	AND A.atttypid = T.oid 
	AND A.attnum > 0;-- 自定义表大于0，系统默认表小于0
