    XtraBackup 相关总结

    # 简介

    基于MySQL服务器的的开源备份软件。

    功能支持：

    • 全量备份

    • 增量备份

    • 传输压缩的备份文件至其他服务器

    优势特点：

    • 备份、还原速度快
    • 备份过程不打断正在执行的事务，即InnoDB热备份
    • 备份检验

    官方文档：官方文档 (opens new window)、参数说明 (opens new window)

    参考博客：数据库备份还原 (opens new window)、XtraBackup参考 (opens new window)

    实际备份

    # 安装运行

    xtrabackup必须和MySQL安装同一服务器，需要能连接交互。

    所以不同形式的安装，对于后面的操作命令不同。

    MySQL版本5.7.x，对应XtraBackup稳定版2.4

    MySQL版本8.0.x，对应XtraBackup稳定版8.0

    # 系统安装

    参考官方文档 (opens new window)

    # 容器安装

    # 安装MySQL

    Docker安装MySQL，可以参考MySQL安装

    需要挂载的目录：

    • 数据库目录，默认配置路径是/var/lib/mysql
    • socket目录，默认配置路径是/var/run/mysqld

    建议也挂载配置文件（目录）

    # 安装XtrBackup

    # docker-compose
version: '3'

services:
  xtrbackup:
    container_name: xtrbackup
    restart: always
    image: bitnami/percona-xtrabackup:2.4
    environment:
     TZ: Asia/Shanghai
    volumes:
      - /ws/soft/xtrabackup/backup:/backup
      - /ws/soft/mysql/data:/mysql/data
      - /ws/soft/mysql/socket:/mysql/socket

    # Docker版本区别

    以全量备份为例。命令在容器内部操作。

    # 最简单命令，在/backup目录下生成备份目录（日期命名）
innobackupex -p 111111yh! /backup
    1
    2

    看官方的参数说明文档，省略以下默认缺省参数：

    主机-H localhost 端口-P 3306 用户名 -u root 配置文件 -default-file /etc/my.cnf，主要读取2个属性

    • 数据库目录datadir=/var/lib/mysql
    • socket文件 socket=/var/run/mysqld/mysqld.sock

    备份目录--backup --target-dir = /data/backups/ 省略为/backup

    因为Docker的文件挂载问题，即使挂载配置文件，读取的属性也是不对的，所以配置属性都要单独指定

    # docker版本最简单命令
innobackupex -P 111111yh! --socket=/mysql/socket/mysqld.sock --datadir=/mysql/data/ /backup
    1
    2

    MySQL的socket和datadir需要挂载到xtrabackup容器内，参考上面的安装

    # 全量备份还原

    以Docker版本介绍命令

      # 备份后，/backup下自动生成目录（日期取名），注意打印的日志
innobackupex  -p 111111yh! --socket=/mysql/socket/mysqld.sock --datadir=/mysql/data/ --databases=tt  /backup

    # 增量备份还原

    以Docker版本介绍命令

    • 第一次，全量备份，参考上面命令

    • 数据库变动，比如增、删、改

    • 增量备份，假设上次备份目录是/backup/2021-08-11_11-05-51

      # --incremental /backup，增量备份的目录也存放在/backup下
# --incremental-basedir=/xxx 指定上个备份目录
innobackupex  -p 111111yh! --socket=/mysql/socket/mysqld.sock --datadir=/mysql/data/ --incremental /backup --incremental-basedir=/backup/2021-08-11_11-05-51

      备份目录下的xtrabackup_checkpoints，描述了备份类型，数据大小等等

    • 还原备份，和上面一样，还原到新库中。准备新库。

    • 检查数据一致性。

      # 第一次的全量备份，除了最后次外都加上 --redo-only参数，即事务不会回滚。
innobackupex --apply-log --redo-only /backup/2021-08-11_11-05-51

    # 其他命令参数

    # 并行备份，即多线程操作，会增加IO
innobackupex -p 111111yh! --parallel=8 /backup

# 节流备份，与上面相反，节省IO，参数表示每秒限制操作IO的次数
innobackupex -p 111111yh! --throttle=200 /backup

# 压缩备份，压缩算法是“quicklz”，后缀名是“.qp"
innobackupex -p 111111yh! --compress /backup
# 压缩线程数量
innobackupex -p 111111yh! --compress --compress-threads=8 /backup
# 解压备份文件，前提安装了qpress
innobackupex --decompress /backup/2021-08-14_11-05-51

# 解压得到qpress，移动到命令行目录下/user/local/bin

qpress -d test.qp ./
# 循环当前目录下.qp文件，解压至当前目录
